## 第 1 题【字符类型转换】(2分) ✗

如果字符变量 `_1` 的值是字符 `'1'`，那么 `(int)_1` 的值是（ ）。

- A. 1
- B. -1
- **C. 49** ← **正确答案**
- D. +1 或者 -1

### 选项分析

| 表达式 | 值 | 说明 |
|:-------|:--:|:-----|
| `(int)'0'` | 48 | 字符零的 ASCII 码 |
| `(int)'1'` | **49** | 字符一的 ASCII 码 |
| `(int)'2'` | 50 | 字符二的 ASCII 码 |

字符 `'1'` 在内存中存储的是其 ASCII 码值 **49**，不是数字 1。强转 `int` 取出该 ASCII 值。

**字符转数字**的正确写法是 `'1' - '0'` → 49 − 48 = **1**（利用数字字符 ASCII 的连续性）。

## 第 9 题【按位取反】(2分) ✗

补码的情况下，关于按位取反运算，以下说法**错误的是**（ ）。

- A. `~5` 的结果是 -6（int 类型，32 位）
- **B. `~0` 的结果是 0（int 类型，32 位）** ← **正确答案**
- C. `~(-3)` 的结果是 2（int 类型，32 位）
- D. `~8` 的结果是 -9（int 类型，32 位）

### 选项分析

补码下按位取反的规律：**`~x = -x - 1`**。

| 选项 | 表达式 | 公式计算 | 二进制验证（32位补码） | 正误 |
|:----:|:-------|:--------:|:---------------------|:----:|
| A | `~5` | -5-1 = **-6** | `~00000000 00000000 00000000 00000101 = 11111111 11111111 11111111 11111010` = -6 | ✅ |
| **B** | **`~0`** | **-0-1 = -1** | **`~00000000 00000000 00000000 00000000 = 11111111 11111111 11111111 11111111` = -1** | **❌ 等于 -1，不是 0** |
| C | `~(-3)` | -(-3)-1 = 2 | `~11111111 11111111 11111111 11111101 = 00000000 00000000 00000000 00000010` = 2 | ✅ |
| D | `~8` | -8-1 = **-9** | `~00000000 00000000 00000000 00001000 = 11111111 11111111 11111111 11110111` = -9 | ✅ |

B 声称 `~0 = 0`，但 `~0 = -1`，故 B 为错误说法。

## 第 11 题【数组与sizeof】(2分) ✗

执行以下代码后，输出结果是（ ）。

```cpp
int arr[] = {5, 10, 15, 20, 25, 30};
int count = sizeof(arr) / sizeof(arr[0]);
cout << count;
```

- A. 4
- **B. 6** ← **正确答案**
- C. 24
- D. 30

### 选项分析

```cpp
sizeof(arr)         = 6 × 4 = 24    // 整个数组的字节数（6个int）
sizeof(arr[0])      = 4             // 一个int的字节数
count = 24 / 4      = 6             // 元素个数
```

`sizeof(arr)/sizeof(arr[0])` 是 C/C++ 计算数组长度的标准写法。

- C 选项（24）是 `sizeof(arr)` 的值，不是 count
- count 是除完之后的 **6**

## 第 12 题【sizeof与strlen】(2分) ✗

执行以下代码后，输出结果是（ ）。

```cpp
char s[10] = "abcde";
int a = sizeof(s) / sizeof(s[0]);
int b = strlen(s);
cout << a - b;
```

- A. 4
- **B. 5** ← **正确答案**
- C. 6
- D. 10

### 选项分析

```
char s[10] = "abcde";
// 内存布局: {'a','b','c','d','e','\0', ?, ?, ?, ?}

sizeof(s)              = 10     // 数组声明容量 10 字节
sizeof(s[0])           = 1      // char 占 1 字节
a = 10 / 1             = 10

strlen(s)              = 5      // 从 s[0] 到 '\0' 前的字符数
b                      = 5

cout << a - b          = 5
```

### sizeof 与 strlen 的区别

| 表达式 | 含义 | 本题值 |
|:-------|:-----|:------:|
| `sizeof(s)` | 数组**声明容量**（字节） | **10** |
| `strlen(s)` | 字符串**实际长度**（不含 `\0`） | **5** |
| `sizeof(s)/sizeof(s[0])` | 数组可容纳元素总数 | **10** |

`strlen` 不计算结尾的 `\0`。

## 第 13 题【枚举法】(2分) ✗

以下问题中，**最不适合**用枚举法解决的是（ ）。

- A. 找出 1~100 之间所有能被 7 整除的数
- B. 找出 100~200 之间的所有质数
- **C. 计算 1+2+3+…+1000 的和** ← **正确答案**
- D. 找出三位数中个位、十位、百位数字之和等于 10 的数

### 选项分析

枚举法的本质：**逐一列举所有候选，逐一判断是否符合条件**。当问题存在直接数学公式时，枚举就是最不合适的方法。

| 问题 | 枚举量 | 是否存在更优方法 | 结论 |
|:-----|:------:|:---------------|:----:|
| A. 1~100 被 7 整除 | 100 次 | 无（仍需逐个检查） | 适合 ✅ |
| B. 100~200 找质数 | 101 次 | 无（仍需逐个判断） | 适合 ✅ |
| **C. 1+2+…+1000** | 1000 次 | **`S = n(n+1)/2` 一步出结果** | **最不适合** ❌ |
| D. 各位之和=10 | 900 次 | 无（仍需逐个检查） | 适合 ✅ |

C 项用等差数列求和公式 `1000×1001/2 = 500500`，O(1) 可得，无需枚举。

## 第 14 题【枚举法-鸡兔同笼】(2分) ✗

用枚举法解决"鸡兔同笼问题：头共 35 个，脚共 94 只，求鸡和兔的数量"，以下枚举逻辑**最合理**的是（ ）。

- **A. 枚举鸡的数量 x（0~35），兔的数量 y=35-x，判断 2*x + 4*y == 94** ← **正确答案**
- B. 枚举兔的数量 y（0~94），鸡的数量 x=35-y，判断 2*x + 4*y == 94
- C. 枚举所有整数 x（0~100）和 y（0~100），判断 x+y==35 && 2*x+4*y==94
- D. 枚举脚的总数 sum（0~94），判断 sum == 94

### 选项分析

合理的枚举应做到：**枚举范围最小、循环层数最少**。

| 方案 | 循环次数 | 层数 | 问题 |
|:----|:--------:|:----:|:-----|
| **A ✅** | **36 次** | **单层** | 利用 `x+y=35` 消元，范围精确 0~35 |
| B | 95 次 | 单层 | 兔数量 y 枚举到 94，但头只有 35，范围冗余 |
| C ❌ | 10201 次 | 双层 | 101×101 无意义枚举，效率最低 |
| D | 95 次 | 单层 | 枚举脚总数无法直接得到鸡兔数量 |

**A 最优的理由**：
1. 用"头共 35"消去一个变量：`y = 35 - x`，从二重降为一重
2. x 范围 0~35（鸡最多 35 只），无冗余
3. 每个候选只需一次判断

## 第 9 题（判断题 第 24 题）(2分) ✗

在 C++ 语言中，表达式 `((0xf0 + 0x15) == 255)` 的值为 true。（ ）

**正确答案：F**

### 分析

```
0xf0 = 15×16 + 0 = 240
0x15 = 1×16 + 5 = 21
240 + 21 = 261

二进制加法验证：
  0xf0 = 1111 0000
+ 0x15 = 0001 0101
------------------
       1 0000 0101 = 0x105 = 261
```

- `0xf0 + 0x15` = **261**，不是 255
- `0xff` 才是 255（`15×16+15 = 255`）
- 表达式值为 `false`，题干说 `true` 是错的
