# 图论与搜索

## NOIP2016 普及组

### T3 海港 (port)

![图](images/noip2016/page07/page07.png)

【题目描述】
小W有一艘货船，每天到达港口卸货。船到港时间ti，带ki种货物编号xi,1~xi,ki。货物在港保留24小时。每天结束时统计当前有多少种货物在港。

输入：第一行n。接下来n行每行ti ki xi,1...xi,ki。
输出：n行当前在港货物种类数。
范围：n≤10^5, sum(ki)≤3×10^5, 1≤ti≤10^9。

【输入格式】
第一行一个整数 n，表示有 n 艘货船到达港口。
接下来 n 行，每行描述一艘货船的信息。第 i 行第一个整数 ti 表示第 i 艘货船到达的时刻，第二个整数 ki 表示这艘货船带来的货物种类数，接下来 ki 个整数 xi,1, xi,2, ..., xi,ki 表示这 ki 种货物的编号。

【输出格式】
输出 n 行，第 i 行一个整数，表示第 i 天结束时（即时刻 ti），当前在港口中的货物种类数。

【样例输入】
```
3
1 4 1 2 2 3
2 2 2 3
10 1 3
```

【样例输出】
```
3
4
4
```

【样例解释】
- 时刻1：船1到达，带来货物{1,2,3}。在港种类数=3。
- 时刻2：船2到达，带来货物{2,3}。货物1已过24小时（时刻1到时刻2不足24小时，保留）。在港种类数=4（货物1,2,3加上新的2,3不增加种类）。
- 时刻10：船3到达，带来货物{3}。货物1在时刻1到达，到时刻10已过9小时<24，保留。在港种类数=4。

【数据范围与约定】
- n ≤ 10^5
- sum(ki) ≤ 3 × 10^5
- 1 ≤ ti ≤ 10^9
- 1 ≤ xi,j ≤ 10^9

## NOIP2017 普及组

### T4 跳房子 (jump)

![图](images/noip2017/page08/page08.png)

【题目描述】
跳房子游戏：起点右侧n个格子，每个格子有分数。机器人每次弹跳距离固定为d。花g金币可增加灵活性（距离范围扩大）。求至少获k分的最小金币数。

【输入格式】
第一行三个整数 n, d, k，分别表示格子数、初始弹跳距离、目标分数。
接下来 n 行，每行两个整数 xi, si，表示第 i 个格子的位置和分数。

【输出格式】
输出一行一个整数，表示最少需要花费的金币数。如果无论如何都无法获得至少 k 分，输出 -1。

【样例输入】
```
7 4 10
2 6
5 -3
10 3
11 -3
13 1
17 6
20 2
```

【样例输出】
```
2
```

【样例解释】
花费2金币后，弹跳距离范围变为[d-g, d+g]=[2, 6]。可以按以下路径跳：位置2(6分) → 位置5(-3分) → 位置10(3分) → 位置13(1分) → 位置17(6分)，总分=6-3+3+1+6=13≥10。

【数据范围与约定】
- n ≤ 500000
- d ≤ 2000
- xi ≤ 10^9
- |si| ≤ 10^5
- k ≤ 10^9

## CSP-J 2023

### T4 旅游巴士 (bus)

![图](images/2023/page09/page09.png)

【题目描述】
n个点m条单向边，边权1。每k时间发一班车。每条边有开放时间ai，不早于ai才能通过。求从1到n的最早离开时间。

【输入格式】
第一行三个整数 n, m, k，分别表示点数、边数和发车间隔。
接下来 m 行，每行三个整数 ui, vi, ai，表示从 ui 到 vi 有一条有向边，该边的开放时间为 ai（即只能在时刻 ≥ ai 时通过该边）。

【输出格式】
输出一行一个整数，表示从点1到点n的最早到达时刻。如果无法到达，输出 -1。

【样例输入】
```
5 5 3
1 2 0
2 5 1
1 3 0
3 4 3
4 5 1
```

【样例输出】
```
6
```

【样例解释】
- 时刻0从点1出发，走边(1,2)到达点2（时刻1）。
- 时刻1从点2出发，走边(2,5)到达点5（时刻2），但边(2,5)的开放时间为1，满足条件。
- 或者：时刻0从点1出发，走边(1,3)到达点3（时刻1）。
- 时刻1从点3出发，等待到时刻3走边(3,4)到达点4（时刻4）。
- 时刻4从点4出发，走边(4,5)到达点5（时刻5），但边(4,5)的开放时间为1，满足条件。
- 最早到达时刻为6（需要考虑班车发车时间的约束）。

【数据范围与约定】
- n ≤ 10^4
- m ≤ 2 × 10^4
- k ≤ 100
- 0 ≤ ai ≤ 10^6

## CSP-J 2024

### T4 接龙 (chain)

![图](images/2024/page11/page11.png)

【题目描述】
n人游戏每人有词库。每轮选[2,k]长连续子序列接龙：第一轮以1开头，后续以上一轮末元素开头，接龙人不能与上一轮相同。q个任务问是否存在恰好r轮且末元素为c的游戏过程。

【输入格式】
输入包含多组测试数据。
第一行一个整数 T，表示测试数据组数。
每组数据的第一行三个整数 n, k, q，分别表示人数、接龙序列长度上限和任务数。
接下来 n 行，每行描述一个人的词库。第 i 行第一个整数 li 表示第 i 个人词库中的元素个数，接下来 li 个整数表示词库中的元素。
接下来 q 行，每行两个整数 r, c，表示一个任务：是否存在恰好 r 轮且末元素为 c 的游戏过程。

【输出格式】
对于每个任务，输出一行。如果存在满足条件的游戏过程，输出 1；否则输出 0。

【样例输入】
```
1
3 3 7
5 1 2 3 4 1
3 1 2 5
3 5 1 6
1 2
1 4
2 4
3 4
6 6
1 1
7 7
```

【样例输出】
```
1
0
1
0
1
0
0
```

【样例解释】
- 任务1 (r=1, c=2)：第1轮，人1选子序列[1,2]，末元素为2。满足，输出1。
- 任务2 (r=1, c=4)：第1轮必须以1开头，人1的词库有1,2,3,4,1，可以选[1,2,3,4]末元素为4。但需要检查是否合法...输出0。
- 任务3 (r=2, c=4)：第1轮人1选[1,2,3]末元素3，第2轮人2以3开头...需要词库中有连续子序列以3开头。输出1。
- 其他任务类似分析。

【数据范围与约定】
- T ≤ 5
- n ≤ 10^5
- 2 ≤ k ≤ 2 × 10^5
- q ≤ 10^5
- 所有词库元素之和 ≤ 2 × 10^5
