### 1、<a href="https://fslong.iok.la/problem/P9202" target="_blank">攒原石</a>(100分)

**题目描述**
小明在攒原石准备抽他心心念念的五星角色。

他每天稳定获得：每日委托 60 + 小月卡 90 = **150 原石/天**。
除此之外，每**连续签到满 7 天**（第 7、第 14、第 21…天）还会额外拿到 **300 原石**的周常奖励。

一次十连抽需要 **1600 原石**。小明现在已有 S 原石。

请你算一算：最少还需要多少天才能凑够一次十连？（不许用循环一天一天地加——试试用数学方法！）

**输入格式**
一行，一个整数 S（0 ≤ S ≤ 10⁵）。

**输出格式**
一行，一个整数。

**样例输入 1**
```
1450
```

**样例输出 1**
```
1
```

**样例解释**
还差 150，每天 150，刚好 1 天。

**样例输入 2**
```
0
```

**样例输出 2**
```
9
```

**样例解释**
7 天为一个周期：每周期 150×7+300=1350。1 个周期后 1350<1600，还差 250。之后每天 150，250÷150 向上取整 = 2 天。共 7+2=9 天。

**提示**
- 关键思路：把 7 天打包成一个"周期"，先看需要几个完整周期
- 周期内不够的部分，再用剩余天数补齐
- 除法 + 向上取整：`(need + 149) / 150`

### 2、<a href="https://fslong.iok.la/problem/P9203" target="_blank">小红书打卡</a>(100分)

**题目描述**
小美参加了小红书的"N天连续打卡挑战"。规则如下：
- 每天只要发帖，就能获得 **10 基础分**
- 如果连续打卡第 k 天（k = 1, 2, 3…），当天额外获得 **5 × k 奖励分**
- 如果某天断签（不发帖），连续天数归零，第二天重新从 k=1 开始算

现在给你 N 天的打卡记录（`1` 表示打卡，`0` 表示断签），请你计算 N 天下来小美一共获得了多少分。

**输入格式**
第一行：一个整数 N（1 ≤ N ≤ 100）。
第二行：N 个整数（每个为 0 或 1），空格分隔，表示每天的打卡情况。

**输出格式**
一行，一个整数，表示总积分。

**样例输入**
```
7
1 1 1 0 1 1 1
```

**样例输出**
```
120
```

**样例解释**
前三天连续：10+5×1 + 10+5×2 + 10+5×3 = 15+20+25 = 60
第四天断签：0 分，连续清零
后三天连续：10+5×1 + 10+5×2 + 10+5×3 = 60
总计：60+60=120

**提示**
- 维护一个 `streak`（连续天数），遇到打卡 streak+1，遇到断签 streak=0
- 每天得分 = 如果打卡：10 + 5×streak；否则 0

### 3、<a href="https://fslong.iok.la/problem/P9204" target="_blank">砍一刀</a>(100分)

**题目描述**
小明的妈妈在拼多多上看中了一件商品，标价 P 元（保留两位小数）。平台提示：邀请好友"砍一刀"，每刀可以砍掉当前价格的 **K%**（砍完后价格向下取整到分）。

注意：每次砍价后，价格会更新，下一刀是在新价格的基础上继续砍 K%。

小明妈妈想知道：从初始价格开始，到底需要砍多少刀，价格才能降到 **0.01 元以下**？

请你模拟这个过程。

**输入格式**
一行，两个数：P（浮点数，0 < P ≤ 10000，保留两位小数）和 K（整数，1 ≤ K ≤ 99）。

**输出格式**
一行，一个整数，表示需要的刀数。

**样例输入 1**
```
100.00 10
```

**样例输出 1**
```
66
```

**样例解释**
每次砍 10%，价格不断折上折：100→90→81→72.9→…→0.01 以下，共 66 刀。

**样例输入 2**
```
50.00 5
```

**样例输出 2**
```
149
```

**提示**
- 价格运算建议转为"分"（整数），避免浮点精度问题：`price = int(P × 100)`
- 每次砍价：`price = price × (100 - K) / 100`（整数除法自动向下取整）

### 4、<a href="https://fslong.iok.la/problem/P9205" target="_blank">圣遗物搭配</a>(100分)

**题目描述**
小明在原神里刷到了 N 个五星圣遗物。每个圣遗物有两个属性：
- **类型** type（1=花，2=羽毛，3=沙漏，4=杯子，5=头冠）
- **暴击伤害** crit（正整数）

小明要选 **恰好 2 个不同类型** 的圣遗物搭配，使得它们的暴击伤害之和最大。请你帮他找出这个最大值。

**输入格式**
第一行：一个整数 N（2 ≤ N ≤ 100）。
接下来 N 行：每行两个整数 type 和 crit（1 ≤ type ≤ 5，1 ≤ crit ≤ 1000），空格分隔。

**输出格式**
一行，一个整数，表示两件不同类型圣遗物的最大暴击伤害之和。
如果不存在两个不同类型的圣遗物，输出 `-1`。

**样例输入**
```
5
1 46
2 62
1 38
3 55
4 71
```

**样例输出**
```
133
```

**样例解释**
枚举所有不同类型对：
- 花(46) + 羽毛(62) = 108
- 花(46) + 沙漏(55) = 101
- 花(46) + 杯子(71) = 117
- 羽毛(62) + 沙漏(55) = 117
- 羽毛(62) + 杯子(71) = **133** ← 最大！
- 沙漏(55) + 杯子(71) = 126

**提示**
- 用双重循环枚举所有圣遗物对 (i, j)，i<j
- 检查 `type[i] != type[j]`，满足则更新最大值

### 5、<a href="https://fslong.iok.la/problem/P9206" target="_blank">朋友圈选片</a>(100分)

**题目描述**
小红今天出去玩了，拍了 N 张照片。每张照片她打了一个"好看指数"（1~10 的整数）。

她想选 **恰好 K 张** 发朋友圈。为了让照片墙整体风格协调，她希望选出的 K 张照片的**平均好看指数尽可能接近目标值 T**（T 是她最喜欢的风格分数，比如 8 代表"高质感冷淡风"）。

请你枚举所有可能的 K 张照片组合，找出**平均分与 T 的差的绝对值最小**的那一组，输出这个最小差值（保留两位小数）。

**输入格式**
第一行：三个整数 N、K、T（2 ≤ K ≤ N ≤ 15，1 ≤ T ≤ 10）。
第二行：N 个整数（范围 1~10），空格分隔，表示每张照片的好看指数。

**输出格式**
一行，一个浮点数，保留两位小数，表示最小差值。

**样例输入**
```
5 3 7
10 8 6 4 2
```

**样例输出**
```
0.33
```

**样例解释**
C(5,3)=10 种组合的平均分：
(10,8,6)=8.00 | (10,8,4)=7.33 | (10,8,2)=6.67 | (10,6,4)=6.67 | (10,6,2)=6.00
(8,6,4)=6.00 | (8,6,2)=5.33 | …
与 7 最接近的是 7.33（差 0.33）、6.67（差 0.33），最小差值为 0.33。

**提示**
- N ≤ 15 意味着 C(15,7)=6435 种组合，枚举完全可以接受
- 可以用多重循环，或自己写一个组合枚举的递归
- 平均值用 `double` 计算，差值 = `|avg - T|`
