### 1、<a href="https://fslong.iok.la/problem/P9018" target="_blank">神秘数字的寻找</a>(100分)

**题目描述**  
小明在数据世界的广场上看到一个古老的石碑，石碑上刻着一道谜题："我是一个两位数（十进制），我的个位和十位数字不同。如果将我的两位数字交换位置，得到的新数与原数的和是99。请问我可能是哪些数字？"

小明需要你编写程序，找出所有符合条件的数字，并按从小到大的顺序输出。

**输入格式**  
无输入。

**输出格式**  
若干行，每行一个整数，表示所有符合条件的数字（从小到大输出）。

**样例输出**  
```
18
27
36
45
54
63
72
81
```

**提示**  
这是一个枚举问题。设原数为 10a+b，交换后为 10b+a，需要满足 (10a+b) + (10b+a) = 99 且 a≠b。可以枚举 a 和 b 的所有可能组合。

### 2、<a href="https://fslong.iok.la/problem/P9017" target="_blank">密码组合破解</a>(100分)

**题目描述**  
小明来到了数据世界的密码库门前。门上有一个三位数的密码锁，密码满足以下条件：

**输入格式**  
无输入。

**输出格式**

**样例输出（部分）**  
```

```

**提示**

### 3、<a href="https://fslong.iok.la/problem/P9019" target="_blank">时间进制转换</a>(100分)

**题目描述**  
数据世界使用一种特殊的时间系统：一天被分为 n 个小时（而不是24小时）。小明需要将十进制表示的时间转换为这个特殊的 n 进制时间格式。

一行，表示转换后的 n 进制时间（如果 n>10，使用字母 A-Z 表示10-35）。

**样例输入**  
```
15 12
```

**样例输出**  
```
13
```

**样例输入 2**  
```
255 16
```

**样例输出 2**  
```
FF
```

### 4、<a href="https://fslong.iok.la/problem/P9020" target="_blank">反向解码器</a>(100分)

**题目描述**  
小明在数据世界的解密室发现了一个反向解码器。这个解码器可以将一个数字的二进制表示"翻转"后，再转换回十进制。

例如：13 的二进制是 1101，翻转后变成 1011，对应的十进制是 11。

小明需要你编写程序，实现这个反向解码功能。

**输入格式**  
一行，一个十进制正整数 n (1 ≤ n ≤ 1000)。

**输出格式**  
一行，一个整数，表示二进制翻转后的十进制值。

**样例输入**  
```
13
```

**样例输出**  
```
11
```

**提示**  
13 的二进制是 1101，翻转后是 1011 = 8+2+1 = 11。

可以先将 n 转换为二进制字符串，翻转字符串，再转换回十进制。

### 5、<a href="https://fslong.iok.la/problem/P9021" target="_blank">信号加密器</a>(100分)

**题目描述**  
数据世界的通信站使用一种特殊的加密方式：将原始信号的每个二进制位取反（0变成1，1变成0），然后传输。小明截获了一段加密信号，需要将其解密回原始信号。

例如：如果加密信号是 5（二进制 101），那么原始信号就是 2（二进制 010）。

小明需要你编写程序，将加密信号解密。

**输入格式**  
一行，一个整数 n (0 ≤ n ≤ 1000)，表示加密信号的十进制值。

**输出格式**  
一行，一个整数，表示解密后的原始信号值。

**样例输入**  
```
5
```

**样例输出**  
```
2
```

**样例输入 2**  
```
13
```

**样例输出 2**  
```
2
```

**提示**  
5 的二进制是 101，按位取反后是 010 = 2。
13 的二进制是 1101，按位取反后是 0010 = 2（只考虑有效位）。

注意：题目中的取反操作只对数字的二进制有效位进行，不包括前导零。可以先确定数字的二进制位数，然后对该位数范围内的所有位取反。

### 6、<a href="https://fslong.iok.la/problem/P9022" target="_blank">数据恢复专家</a>(100分)

**题目描述**  
小明来到了数据世界的恢复中心。这里有一批损坏的数据文件，每个文件都存储了两个数字 a 和 b 的信息。不幸的是，文件只保存了 a+b 和 a^b（a与b的异或）的值，原始数据丢失了。

恢复中心的工程师说："如果你能根据 a+b 和 a^b 还原出 a 和 b，就能恢复这些文件。"

小明需要你编写程序，给定 sum = a+b 和 xor = a^b，找出满足条件的 a 和 b（如果有多组解，输出 a 最小的那组）。

**输入格式**  
一行，两个整数 s 和 x (0 ≤ s, x ≤ 10000)，分别表示 a+b 和 a^b 的值。

**输出格式**  
如果存在解，输出一行两个整数 a 和 b（a ≤ b）；否则输出 "NO SOLUTION"。

**样例输入**  
```
9 5
```

**样例输出**  
```
2 7
```

**样例输入 2**  
```
10 3
```

**样例输出 2**  
```
NO SOLUTION
```

**提示**  
利用位运算的性质：a + b = (a ^ b) + 2 * (a & b)

所以 a & b = (s - x) / 2。如果 s-x 是负数或奇数，则无解。

已知 a^b 和 a&b 后，可以通过枚举或数学方法求出 a 和 b。
