### 1、P9431 订单确认（100 分）

**题目描述**  
高橋君是餐厅的楼面服务员。今天有 N 位客人各点了一份菜。  
第 i 位客人**实际点的菜名**是 T_i，但高橋君**向厨房传达的菜名**是 S_i。  
请统计有多少位客人的订单被传达错了（即 T_i ≠ S_i 的人数）。

**输入格式**  
第一行 N（1 ≤ N ≤ 10⁵）。  
接下来 N 行，每行两个由小写字母组成的字符串 T_i 和 S_i（长度 1~20）。

一个整数，表示传达错误的订单数。

**样例**
```
输入：
4
ramen ramen
sushi susi
curry curry
pasta pizza

输出：
2
```

**提示** 直接比较字符串即可。

**知识点**：字符串比较、循环计数

### 2、P9432 击杀怪物（100 分）

**题目描述**  
RPG 游戏中，N 只怪物从左到右排列。第 i 只怪物强度为 D_i，击败可得 V_i 金币。  
勇士初始攻击力 h = 0。击败怪物后，攻击力增加 D_i。  

勇士有一只"后取栈"（LIFO 栈）。依次访问每只怪物：
- 若 D_i > h（攻击力不足）：将该怪物压入栈顶。
- 若 D_i ≤ h：击败之，获得 V_i，攻击力增加 D_i。然后尝试**消栈**：
  - 只要栈非空且栈顶怪物强度 ≤ h，就弹出并击败之（获得金币、增加攻击力）。

访问完所有怪物后，再做一次消栈操作。最终栈中未击败的怪物不获得金币。  
求获得金币总数。

**输入格式**  
第一行 N（1 ≤ N ≤ 2×10⁵）。  
接下来 N 行，每行 D_i V_i（0 ≤ D_i ≤ 10⁹，1 ≤ V_i ≤ 10⁹）。

**输出格式**  
一个整数，表示总金币数。

**样例**
```
输入：
5
0 10
3 20
1 15
0 5
2 30

输出：
15
```

**知识点**：栈、模拟

**提交链接**：<a href="https://fslong.iok.la/problem/9432" target="_blank">P9432 - 击杀怪物</a>

### 3、P9433 植物生长记录（100 分）

高橋君种了 N 棵植物。第 i 棵当前高度 A_i mm，每天生长 B_i mm。  
问 D 天后，所有植物高度之和是多少？答案不超过 2×10¹⁶。

**输入格式**  
第一行 N D（1 ≤ N ≤ 10⁵，1 ≤ D ≤ 10⁶）。  
接下来 N 行，每行 A_i B_i（1 ≤ A_i ≤ 10⁹，1 ≤ B_i ≤ 10⁵）。

**样例**
```
输入：

10 2

75
```

**提示** 每棵植物 D 天后高度 = A_i + B_i × D，全部求和。注意用 long long。

### 4、P9429 炼金配方（100 分）

**题目描述**  
在魔法世界里，炼金术士小杨想要炼制一种 potency 值为 n 的魔法药水。他有 k 种不同的魔法材料，每种材料 i 的 potency 值为 p[i]。小杨可以按任意顺序依次添加材料，每次添加一种，材料的 potency 值会累加到药水中。

小杨想知道：有多少种不同的添加序列，能使总 potency 恰好等于 n？

注意：不同的添加顺序算不同的序列。例如，添加材料 A 再添加材料 B（A→B）和添加 B 再添加 A（B→A）是两种不同的序列。

由于答案可能很大，请输出对 10⁹+7 取模的结果。

**输入格式**  
第一行两个整数 n 和 k（1 ≤ n ≤ 1000, 1 ≤ k ≤ 20）。  
第二行 k 个整数 p[1..k]（1 ≤ p[i] ≤ n），表示每种材料的 potency 值。

**输出格式**  
一行，一个整数，表示不同的添加序列数，对 10⁹+7 取模。

**样例输入**
```
5 3
1 2 3
```

**样例输出**
```
13
```

**样例解释**  
所有可能的序列（共 13 种）：
```
1: 1+1+1+1+1
2: 1+1+1+2   3: 1+1+2+1   4: 1+2+1+1   5: 2+1+1+1
6: 1+1+3     7: 1+3+1      8: 3+1+1
9: 1+2+2    10: 2+1+2     11: 2+2+1
12: 2+3      13: 3+2
```

**提示**  
定义 `dfs(x)` 表示凑出 potency 值 x 的序列数。
- 边界条件：`dfs(0) = 1`（什么都不加就是一种方式）
- 转移：`dfs(x) = sum(dfs(x - p[i]))`（对所有 p[i] ≤ x 的 i）
- 用 `memo[x]` 记录已算过的值

**知识点**：DFS、一维记忆化搜索、计数DP

**提交链接**：<a href="https://fslong.iok.la/problem/9429" target="_blank">P9429 - 炼金配方</a>

### 5、P9430 魔法迷宫（100 分）

**题目描述**  
小勇闯进了一个 n 行 m 列的魔法迷宫。每个格子里都有一块魔法石，第 i 行第 j 列的魔法石能量值为 a[i][j]。

小勇从左上角 (1,1) 出发，每次可以向右、向下或向右下（对角线）移动一格，最终要到达右下角 (n,m)。他走过的路径上所有魔法石的能量值之和，就是他的总得分。

小勇想知道：从 (1,1) 到 (n,m) 的所有路径中，能获得的最大总得分是多少？

**输入格式**  
第一行两个整数 n 和 m（1 ≤ n, m ≤ 100）。  
接下来 n 行，每行 m 个整数 a[i][j]（0 ≤ a[i][j] ≤ 1000），表示每个格子的能量值。

**输出格式**  
一行，一个整数，表示最大总得分。

**样例输入**
```
3 4
1 3 1 5
2 8 0 2
4 1 6 1
```

**样例输出**
```
20
```

**样例解释**  
最优路径：(1,1)→(1,2)→(2,2)→(3,2)→(3,3)→(3,4)，得分 = 1+3+8+1+6+1 = 20

其他路径参考：
- (1,1)→(1,2)→(2,2)→(3,3)→(3,4) = 1+3+8+6+1 = 19
- (1,1)→(2,1)→(2,2)→(2,3)→(3,3)→(3,4) = 1+2+8+0+6+1 = 18

**提示**  
定义 `dfs(i, j)` 表示从 (i,j) 走到 (n,m) 能获得的最大得分。
- 边界条件：`dfs(n, m) = a[n][m]`
- 转移：`dfs(i, j) = a[i][j] + max(dfs(i+1, j), dfs(i, j+1), dfs(i+1, j+1))`
- 用二维 `memo[i][j]` 记录已算过的值

**知识点**：DFS、二维记忆化搜索、二维数组

**提交链接**：<a href="https://fslong.iok.la/problem/9430" target="_blank">P9430 - 魔法迷宫</a>
