### 4、<a href="https://fslong.iok.la/problem/P9227" target="_blank">选数凑和</a>（100 分）

**故事背景**  
小明在爆破手训练营通过了基础试炼，教官给了他一道进阶挑战：从 1 到 n 中**恰好**选出 k 个**不同**的数，使它们的和恰好等于 m。小明发现这题融合了"组合"和"部分和"两种技巧——需要同时约束个数与和值！

**题目描述**  
从 1,2,...,n 中选出恰好 k 个不同的正整数，使得这些数的和等于 m。输出所有方案。

要求：方案内**从小到大**排列，方案间**字典序**输出。

**输入**  
一行，三个整数 n、k、m（1 ≤ k ≤ n ≤ 15，1 ≤ m ≤ 100）。

**输出**  
每行一个方案，数字间用空格隔开。如果无解，输出空行。

**样例输入 1**
```
5 2 6
```

**样例输出 1**
```
1 5
2 4
```

**样例输入 2**
```
6 3 10
```

**样例输出 2**
```
1 3 6
1 4 5
2 3 5
```

**提示**  
- 定义递归函数 `void dfs(int cnt, int sum, int start)`
- `cnt`：已经选了几个数
- `sum`：当前已选数的和
- `start`：当前可从哪个数开始选（保证不重复、递增）
- 终止条件：`cnt == k` 时检查 `sum == m`，若相等则输出
- 递归传递：`dfs(cnt + 1, sum + i, i + 1)` —— 选 `i`，计数+1，和增加，下一个从 `i+1` 开始
- **剪枝**：若 `sum + i > m` 可直接 break（因为后面的数更大，和只会更大）
- **为什么无需回溯？** 每个数只选一次（传 `i+1`），路径独立

数据范围：1 ≤ n ≤ 15，1 ≤ k ≤ n，1 ≤ m ≤ 100

### 5、<a href="https://fslong.iok.la/problem/P9228" target="_blank">每位不重复的 m 进制数</a>（100 分） 

**故事背景**  
小明轻松搞定了前几道爆搜题，教官微微一笑："前面的题都不需要回溯——因为选择要么无限，要么路径独立。但如果每位数字**不能重复**呢？"小明一愣：选了 1 之后就不能再选 1 了，这意味着**选择互斥**——需要 `flag` 数组标记，还需要在递归返回时**恢复现场**！这就是下讲"时光机"要深入的内容，现在先来热个身。

**题目描述**  
生成长度为 n 的 m 进制数，要求**每位数字不能重复**（即 0 ~ m-1 每个数字最多用一次）。输出所有方案。

**输入**  
一行，两个整数 n 和 m（1 ≤ n ≤ m ≤ 6）。

**输出**  
每行一个方案，数字间空格隔开。共 P(m,n) = m!/(m-n)! 行，按字典序输出。

**样例输入 1**
```
2 3
```

**样例输出 1**
```
0 1
0 2
1 0
1 2
2 0
2 1
```

**样例输入 2**
```
3 3
```

**样例输出 2**
```
0 1 2
0 2 1
1 0 2
1 2 0
2 0 1
2 1 0
```

**提示**  
- 定义递归函数 `void dfs(int k)`，`k` 表示当前填到第几位（从 0 开始）
- **需要 `bool flag[10]` 数组**：`flag[i] = true` 表示数字 `i` 已经被用过
- 每层循环：`for (int i = 0; i < m; i++)`，检查 `if (flag[i]) continue;`
- 选中 `i` 后：先 `flag[i] = true`，再 `dfs(k + 1)`，**最后 `flag[i] = false` 恢复现场**
- 终止条件：`k == n`，输出已填的 n 位数字，return
- 总方案数：P(m,n)，可用来验证输出行数
- **为什么需要回溯？** 选择互斥——选了 1 就不能再选 1，需要标记；但当这条路径走完后返回上层时，必须把标记清除，让其他分支还能选 1

数据范围：1 ≤ n ≤ m ≤ 6
