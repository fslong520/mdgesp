今天我们学习的是《信奥数学-乘法原理》，本次课的教学目的有：

1. 理解乘法原理的概念——分步计数、各步相乘
2. 能将分步计数的应用题转化为代码实现（数组输入 + 循环累乘）
3. 区分加法原理（分类→相加）与乘法原理（分步→相乘）
4. 多组测试、幂运算等进阶应用

## 一、重点知识

### 什么是乘法原理？

**乘法原理**：做一件事有 n 步，第 i 步有 m[i] 种不同方法，完成这件事共有 N = m[1] × m[2] × ... × m[n] 种方法。本质就是分步计数、各步相乘。

**加法原理 vs 乘法原理**——最核心的区别是"分类"还是"分步"：
- 加法原理：几类办法，**任选一类**就能完成 → 类类相加
- 乘法原理：几个步骤，**每步都要做**才能完成 → 步步相乘

> 判断口诀："或"用加法，"且"用乘法。比如"吃面或吃饭"是二选一，用加法；"先吃面再喝汤"是两步都要做，用乘法。

**通用代码框架**——和加法原理对称，把 + 换成 ×、int 换成 long long：

```cpp
long long ans = 1;
for (int i = 0; i < n; i++) {
    cin >> a[i];
    ans *= a[i];
}
cout << ans;
```

### 如何用乘法原理解题：三步法

拿到一道题，按三步走：

**第一步：拆步骤**——完成这件事需要依次做哪几件事？用"先...再...然后..."来描述。

**第二步：算每步**——每一步分别有多少种选择？注意每步的选择数可能相互影响（选了一个就少一个）。

**第三步：乘起来**——各步选择数连乘。写代码前先估算乘积范围，决定用 int 还是 long long。

### 从固定步数到任意步数

**P9509 毕业旅行**：起点→A(a条路)、A→B(b条路)、B→终点(c条路)。三步：先从起点到A，再从A到B，最后从B到终点，每步选择独立，方案数 = a × b × c。

```cpp
int a, b, c;
cin >> a >> b >> c;
cout << a * b * c;
```

跟上节课加法原理第一层完全对称，孩子直接迁移，一遍 AC。

**P9501 先进集体评选**也是同样模型，但每步选项数递减——选完一个先进，剩余可选班级就少一个：学习先进 n 种、体育先进 n−1 种、卫生先进 n−2 种。这里拆步也是三步，但每步的选择数依赖于上一步的结果，乘的时候不能都乘 n。孩子识别出了这个变化，一遍过。

到了 **P9502 服装搭配**，问题变成：帽子 a 顶、上衣 b 件、裤子 c 条、鞋子 d 双，各选一件搭配。拆步就是四步，但帽子这步有个变化——可戴可不戴，所以帽子的选择数是 a+1（戴其中一顶或什么都不戴）而不是 a。一开始孩子没注意到"不戴也是一种选择"，后来读题看了样例解释慢慢想明白了。这是个易漏点——判断每步的选择时，要留意有没有"什么都不选"这种隐含选项。

**P9510 抽奖活动**：n 类奖品每类 a[i] 个，每类各取一个。步数 n 不固定，用数组 + 循环累乘。但这次孩子没想到用 long long，乘积超 int 范围炸了——可见溢出意识还不够稳定，看到乘法应该条件反射先估算最大值范围。

### 几步常见易错场景

用乘法原理做题，这几个坑容易踩：

① **漏了「不选」这种隐含选项**——P9502 的帽子可戴可不戴，少算一种。
② **忘记 long long**——乘积超出 int 范围，尤其在步数多或每步选择数大时。
③ **分步和分类混淆**——"去图书馆可以坐公交或地铁"是二选一，用加法不是乘法。
④ **递减相乘时全乘同一个数**——每步选择数可能递减（如 P9501 选先进），不能想当然 n×n×n。

### P9503 字母染色——最有价值的调试题

n 种颜色给 m 个字母染色，每个字母颜色不同，本质是 n×(n−1)×...×(n−m+1)，每染一个字母可选颜色就少一种。

过程：74 分 ×2 → AC → 又 74 分 → 最终 AC。反复 5 次，根源还是乘积超 int 范围，没开 long long——跟 P9510 同一个问题。明明知道要用 long long，做题时却忘了，说明还没形成"看到乘法先估范围"的条件反射。

不过最终自己调出来了，这比一遍 AC 更有收获——实际做题 debug 才是真正巩固知识的过程。
