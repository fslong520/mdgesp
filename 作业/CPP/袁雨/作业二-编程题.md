### 1、<a href="https://fslong.iok.la/problem/P9472" target="_blank">报数出列</a>(100分)

**题目描述**

体育课上，n 个同学站成一圈，从第 1 个同学开始报数：

- 报数从 1 开始；
- 报到 m 的同学出列；
- 下一位同学从 1 开始重新报数；
- 直到所有同学都出列为止。

小雨想知道出列的顺序是什么。请你帮帮她。

**输入格式**

一行两个整数 n 和 m（1 ≤ n ≤ 1000，1 ≤ m ≤ 100）。

**输出格式**

一行 n 个整数，表示出列顺序中每位同学的编号（编号从 1 到 n），用空格隔开。

**样例输入 1**
```
6 5
```

**样例输出 1**
```
5 4 6 2 3 1
```

**样例解释 1**

6 个同学，报到 5 的出列：

- 报数：1→2→3→4→**5**（5号出列）
- 继续：6→1→2→**3**→4（4号出列？等等让我重新算）

实际上过程为：1-2-3-4-5（5号出列），6-1-2-3-4（4号出列），6-1-2-3-6（6号出列），1-2-1-2（2号出列），1-3-1（3号出列），最后1号出列。

所以顺序为 5 4 6 2 3 1。

**样例输入 2**
```
5 1
```

**样例输出 2**
```
1 2 3 4 5
```

**样例解释 2**

报到 1 就出列，所以按顺序逐个出列。

**数据范围与提示**

- 30% 数据：n ≤ 10
- 30% 数据：n ≤ 100
- 40% 数据：n ≤ 1000

> **提示**：可以用一个数组或布尔数组标记已出列的同学。用一个指针模拟报数过程，每步移动 m-1 个未被标记的人，标记当前位置为出列。或者用队列模拟：让队首出队再入队，重复 m-1 次，第 m 个直接出列（不入队）。

### 2、<a href="https://fslong.iok.la/problem/P9473" target="_blank">机器人行走</a>(100分)

**题目描述**

小雨的机器人从原点 (0, 0) 出发，面朝北方（正 y 方向）。机器人接收一串指令：

- `F`：向前走 1 步
- `L`：左转 90 度（不移动）
- `R`：右转 90 度（不移动）

请编程计算机器人执行完所有指令后的最终坐标。

方向说明：
- 初始面向：北（正 y 方向）
- L 左转：北→西，西→南，南→东，东→北
- R 右转：北→东，东→南，南→西，西→北

**输入格式**

一行一个字符串 s，由大写字母 F、L、R 组成（1 ≤ |s| ≤ 1000）。

**输出格式**

一行两个整数 x 和 y，表示最终坐标，中间用空格隔开。

**样例输入 1**
```
FFRFF
```

**样例输出 1**
```
2 2
```

**样例解释 1**

初始 (0,0) 面朝北。F→(0,1)，F→(0,2)，R→朝东，F→(1,2)，F→(2,2)。最终 (2,2)。

**样例输入 2**
```
FLFLFLFL
```

**样例输出 2**
```
0 0
```

**样例解释 2**

F→(0,1)北，L→朝西，F→(-1,1)西，L→朝南，F→(-1,0)南，L→朝东，F→(0,0)东，L→朝北，F→(0,1)北，L→朝西，F→(-1,1)……走了一个正方形回到原点附近。实际上走4步F回到(0,0)。

**数据范围与提示**

- 30% 数据：字符串只含 F
- 30% 数据：|s| ≤ 10
- 40% 数据：|s| ≤ 1000

> **提示**：用一个变量 d 表示方向（0=北，1=东，2=南，3=西）。方向数组 dx[4] = {0, 1, 0, -1}，dy[4] = {1, 0, -1, 0}。遇到 L 则 d = (d + 3) % 4（左转），遇到 R 则 d = (d + 1) % 4（右转），遇到 F 则 x += dx[d], y += dy[d]。

### 3、<a href="https://fslong.iok.la/problem/P9474" target="_blank">最小公倍数</a>(100分)

**题目描述**

小雨在数学课上学了最小公倍数（LCM）的概念：两个或多个整数的最小公倍数是能被它们各自整除的最小正整数。

例如 4 和 6 的最小公倍数是 12（12 能被 4 和 6 整除），6 和 8 的最小公倍数是 24。

现在给定 n 个正整数，请你帮小雨求出它们的最小公倍数。

**输入格式**

第一行一个整数 n（1 ≤ n ≤ 10）。
第二行 n 个正整数 a_1, a_2, ..., a_n（1 ≤ a_i ≤ 1000）。

**输出格式**

一行一个整数，表示这些数的最小公倍数。数据保证结果在 int 范围内。

**样例输入 1**
```
3
4 6 8
```

**样例输出 1**
```
24
```

**样例解释 1**

4=2²，6=2×3，8=2³。LCM 取每个质因子的最高次幂：2³×3=24。

**样例输入 2**
```
2
12 18
```

**样例输出 2**
```
36
```

**数据范围与提示**

- 30% 数据：n = 2，a_i ≤ 20
- 30% 数据：n ≤ 5，a_i ≤ 100
- 40% 数据：n ≤ 10，a_i ≤ 1000

> **提示**：两数的最小公倍数可以用公式 LCM(a,b) = a / GCD(a,b) × b 计算。GCD 用辗转相除法。多个数的最小公倍数：先求前两个的 LCM，再用结果与第三个数求 LCM，依此类推。

### 4、<a href="https://fslong.iok.la/problem/P9475" target="_blank">分糖果</a>(100分)

**题目描述**

小雨有 n 颗糖，要分给若干小朋友。每个小朋友的编号恰好是 n 的因数（但不包括 n 本身），第 d 号小朋友分 d 颗糖。

小雨想知道：糖够不够分？

- 如果 n 恰好等于所有真因数之和，糖刚好分完，输出 `perfect`
- 如果 n 小于真因数之和，糖不够分，输出 `abundant`
- 如果 n 大于真因数之和，糖有剩余，输出 `deficient`

**输入格式**

一行一个正整数 n（1 ≤ n ≤ 10^6）。

**输出格式**

一行一个字符串：`perfect`、`abundant` 或 `deficient`。

**样例输入 1**
```
6
```

**样例输出 1**
```
perfect
```

**样例解释 1**

6 的真因数：1、2、3。和 = 1+2+3 = 6。刚好分完。

**样例输入 2**
```
12
```

**样例输出 2**
```
abundant
```

**样例解释 2**

12 的真因数：1、2、3、4、6。和 = 16 > 12。糖不够分。

**样例输入 3**
```
8
```

**样例输出 3**
```
deficient
```

**样例解释 3**

8 的真因数：1、2、4。和 = 7 < 8。糖有剩余。

**数据范围与提示**

- 30% 数据：n ≤ 100
- 30% 数据：n ≤ 10000
- 40% 数据：n ≤ 10^6

> **提示**：遍历 1 到 √n 找因数。如果 i 是 n 的因数且 i × i ≠ n，则 i 和 n/i 都是真因数。注意 1 没有真因数（和为0），是 deficient。

### 5、<a href="https://fslong.iok.la/problem/P9476" target="_blank">字符串压缩</a>(100分)

**题目描述**

小雨想发明一种简单的压缩方法：将字符串中连续相同的字符替换为"字符+重复次数"。

例如 `AAABBBBC` 压缩为 `A3B4C1`（A 连续 3 次、B 连续 4 次、C 连续 1 次）。

注意：连续出现 1 次也要带数字 1。

请你帮小雨实现这个压缩算法。

**输入格式**

一行一个由大写字母组成的字符串 s（1 ≤ |s| ≤ 1000）。

**输出格式**

一行字符串，表示压缩后的结果。

**样例输入 1**
```
AAABBBBC
```

**样例输出 1**
```
A3B4C1
```

**样例输入 2**
```
ABCD
```

**样例输出 2**
```
A1B1C1D1
```

**数据范围与提示**

- 30% 数据：相邻字符均不相同
- 30% 数据：|s| ≤ 100
- 40% 数据：|s| ≤ 1000

> **提示**：遍历字符串，用变量记录当前字符和出现次数。每当遇到新字符时，将前一个字符及其次数拼到结果中，然后重置计数。

### 6、<a href="https://fslong.iok.la/problem/P9477" target="_blank">字符串周期</a>(100分)

**题目描述**

小雨发现有些字符串有"周期"规律——整个字符串可以由某个较短的子串重复若干次得到。

例如 `ABCABCABC` 由 `ABC` 重复 3 次得到，`AAAA` 由 `A` 重复 4 次得到，最小周期分别为 3 和 1。

而 `ABCDE` 的最小周期就是它自己，长度为 5。

请你帮小雨找出一个字符串的最小周期长度。

**输入格式**

一行一个字符串 s，由大写字母组成（1 ≤ |s| ≤ 1000）。

**输出格式**

一行一个整数，表示最小周期长度。

**样例输入 1**
```
ABCABCABC
```

**样例输出 1**
```
3
```

**样例解释 1**

`ABC` 重复 3 次得到整个字符串，周期长度为 3。

**样例输入 2**
```
AAAA
```

**样例输出 2**
```
1
```

**样例输入 3**
```
ABCDE
```

**样例输出 3**
```
5
```

**数据范围与提示**

- 30% 数据：|s| ≤ 10
- 30% 数据：|s| ≤ 100
- 40% 数据：|s| ≤ 1000

> **提示**：最小周期长度一定是字符串总长度的约数。枚举所有可能的周期长度 len（从 1 到 n），如果 n % len == 0，就检查每 len 个字符是否与第一组完全相同：检查 s[0:len] 重复 n/len 次是否等于 s。
