## 一、重点知识

### 1. 电子时钟（P9466）

**题意**：给定当前时间 h:m:s 和经过的秒数 t，求经过 t 秒后的时间。

**孩子代码**：
```cpp
cin >> h >> m >> s >> t;
s = s + t;
if (s >= 60) { m = m + s / 60; s = s % 60; }
if (m >= 60) { h = h + m / 60; m = m % 60; }
if (h >= 24) { h = h % 24; }
cout << h << ' ' << m << ' ' << s;
```

**评价**：一遍 AC，代码简洁。思路清晰——从最小单位秒开始进位，逐级向分、时推进，最后做 24 小时取模。初学者常犯「先统一转成秒再转回来」的绕路写法，孩子直接用逐级进位，简单直接。

### 2. 打印月历（P9467）

**题意**：输入年份 y 和月份 m，打印该月的日历。

**孩子 AC 代码（749B）**：
```cpp
int a[13] = {0,31,28,31,30,31,30,31,31,30,31,30,31};

// 闰年判断
if (y % 400 == 0) a[2] = 29;
else if (y % 100 != 0 and (y % 4 == 0)) a[2] = 29;

// 从 1900 年算到目标年的前一年，累计天数
for (int i = 1900; i < y; i++) {
    if ((i % 400 == 0) or (i % 100 != 0 and i % 4 == 0)) t += 366;
    else t += 365;
}

// 加上目标年前 m-1 个月的天数
for (int i = 1; i < m; i++) t += a[i];

t %= 7;
t = (t + 1) % 7;  // 1900-01-01 是星期一

// 打印日历
for (int i = 0; i < t + a[m]; i++) {
    if (i < t) cout << "   ";
    else { cout.width(2), cout << i - t + 1 << ' '; }
    if ((i + 1) % 7 == 0) cout << '\n';
}
```

**调试过程**：

| 版本 | 结果 | 原因 |
|:----:|:----:|:------|
| 99496-99498 | **CE** ×3 | 语法错误（likely 拼写/符号问题） |
| 99499 | **33 分** | 首日定位或格式问题 |
| **99501** | **100 分** ✅ | 修正完成 |

**评价**：日历打印是经典模拟题，思路标准——先算从 1900 年到目标年首日的天数再推算星期几。31 行的 `t=(t+1)%7` 源于 1900-01-01 是星期一，做了偏移。三次 CE 是语法不熟的表现，应提醒提交前先本地编译检查。

### 3. 素数统计（P9468）

**题意**：统计区间 [l, r] 内的素数个数。

**孩子 AC 代码（315B）**：
```cpp
bool f[11451411];  // 标记数组
fill(f + 2, f + r + 1, 1);
for (long i = 2; i <= r; i++) {
    if (f[i] == 1) {
        if (i >= l) c++;
        for (long j = i * i; j <= r; j += i) f[j] = 0;
    }
}
cout << c;
```

**调试过程**：

| 版本 | 结果 | 原因 |
|:----:|:----:|:------|
| 99507 | **CE** | 语法问题 |
| 99508 / 99511 / **99513** | **100 分** ✅ | CE 修正后连续 AC |

**评价**：用埃拉托斯特尼筛法，思路正确。数组大小 `11451411` 是梗数字（114514），约 11MB，放在全局区没问题。循环从 `i=2` 开始，没有 Chen Yunxi 那样从 0 开始的浪费——此处写得更好。注意 `j = i * i` 可能溢出 long，不过本题数据范围较小，无影响。

### 4. 颜色代码转换（P9469）

**题意**：将六位十六进制颜色码（如 `FFAABB`）转换为 RGB 十进制值。

**40 分版本（有 bug）**：
```cpp
string d = "0123456789ABCDEF";
cin >> s;
a = d.find(s[0]); b = d.find(s[1]);
cout << a*16 + b*1 << ' ';
a = d.find(s[2]); b = d.find(s[3]);
cout << a*16 + b*1 << ' ';
a = d.find(s[4]); b = d.find(s[5]);
cout << a*16 + a*1 << ' ';    // ❌ 应该是 b*1，写成 a*1！
```

**⚠️ Bug 分析**：第三组 RGB 的第二个十六进制数取了 `a` 而非 `b`——典型的**复制粘贴后漏改变量名**。抄了第一组的代码改了索引，但最后一个变量 `b→a` 忘改了。

**100 分版本（修正）**：
```cpp
a = (s[0] >= 'A' ? s[0] - 'A' + 10 : s[0] - '0');
b = (s[1] >= 'A' ? s[1] - 'A' + 10 : s[1] - '0');
cout << a * 16 + b * 1 << ' ';
// ... 后两组同理，修正为 b*1 ✅
```

**调试过程**：

| 版本 | 结果 | 原因 |
|:----:|:----:|:------|
| 99517-99518 | **40 分** | 第三组 `b*1` 写成 `a*1` |
| 99519 | **100 分** | 删了 `d.find()`，改为 ASCII 运算，修正 bug |
| 99521 | **70 分** | 可能改了但还有残留问题 |
| **99522** | **100 分** ✅ | 最终版本 |

**评价**：孩子用了 `d.find()` 转十六进制，写法优雅。但 40 分版的 bug 是典型复制粘贴漏改——第三行 `a*16+b*1` 写成 `a*16+a*1`。后来自己发现并修正，还换了更高效的 ASCII 算术写法。这题锻炼了**复制粘贴后逐变量复查**的意识。

### 5. 爷爷的密码本（P9470）

**题意**：对字符串做凯撒密码解密，每个字母向后（或向前）偏移 k 位。

**22 分版本（有 bug）**：
```cpp
for (auto e : s) {
    if (e >= 'a' and e <= 'z') { e = e - k; }
    if (e >= 'A' and e <= 'Z') { e = e - k; }
    cout << e;
}
```

**⚠️ Bug 分析**：没有处理**环绕**——当 `e - k < 'a'` 或 `< 'A'` 时，字符会越出字母范围变成其他符号。

**100 分版本（修正）**：
```cpp
for (auto e : s) {
    if (e >= 'a' and e <= 'z') {
        e = e - k;
        if (e < 'a') e = e + 26;    // ✅ 环绕处理
    }
    if (e >= 'A' and e <= 'Z') {
        e = e - k;
        if (e < 'A') e = e + 26;    // ✅ 环绕处理
    }
    cout << e;
}
```

**调试过程**：

| 版本 | 结果 | 原因 |
|:----:|:----:|:------|
| 99526 | **22 分** | 凯撒偏移无环绕处理，越界字母变乱码 |
| 99528-99529 | **100 分** ✅ | 添加 `if(e<'a') e+=26` 和 `if(e<'A') e+=26` |

**评价**：22 分版的 bug 是典型的「边界遗漏」——只考虑了正常的偏移，没有考虑需要回绕的情况。修正后一次过两道 AC，说明思路本身通了，缺的只是边界意识。

### 模拟题的通用方法论

| 步骤 | 内容 |
|:----:|------|
| ① | 读懂题意，明确输入输出 |
| ② | 拆解操作步骤，画出流程图 |
| ③ | 从最简单的子任务开始实现 |
| ④ | 逐子任务验证，发现问题及时修正 |
| ⑤ | 全部通过后检查边界情况 |
