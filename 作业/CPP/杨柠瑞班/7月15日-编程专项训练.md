### 第 1 题 连续打卡红包 ★ [P9732]（100 分）— 模拟 <a href="https://fslong.iok.la/problem/P9732" target="_blank">🧑‍💻 在线答题</a>

【题目描述】

冯老师参加了学校的"步数打卡得红包"活动：每天步数达到 8000 步，即可领一个红包。

不过，红包数量不是固定的，它与冯老师的**连续达标天数**有关：

- 连续第 1~2 天：每天领 **1** 个
- 连续第 3~4 天：每天领 **2** 个
- 连续第 5 天及以上：每天领 **3** 个

如果某天没达标（步数 < 8000），连续天数重新从 0 开始计算。

请你算一算，这 n 天冯老师一共领到了多少个红包？

【输入格式】

第一行，一个整数 n（1 ≤ n ≤ 100）。

第二行，n 个非负整数，表示每天的步数（0 ≤ 步数 ≤ 50000）。

【输出格式】

一行，一个整数，表示领到的红包总数。

【样例 1 输入】

```
8
8000 8000 8000 8000 8000 8000 8000 8000
```

【样例 1 输出】

```
18
```

【样例 1 解释】

连续 8 天达标，每天的红包数随连续天数增加：

| 天数 | 连续第几天 | 当天红包 |
|------|-----------|---------|
| 第 1 天 | 1 | 1 个 |
| 第 2 天 | 2 | 1 个 |
| 第 3 天 | 3 | 2 个 |
| 第 4 天 | 4 | 2 个 |
| 第 5 天 | 5 | 3 个 |
| 第 6 天 | 6 | 3 个 |
| 第 7 天 | 7 | 3 个 |
| 第 8 天 | 8 | 3 个 |

总计 1+1+2+2+3+3+3+3 = **18** 个。

【样例 2 输入】

```
5
8000 8000 7000 8000 8000
```

【样例 2 输出】

```
4
```

【样例 2 解释】

第 1 天达标（连续第 1 天）→ 1 个
第 2 天达标（连续第 2 天）→ 1 个
第 3 天步数 7000 < 8000 → **不达标**，连续天数清零
第 4 天达标（连续第 1 天）→ 1 个
第 5 天达标（连续第 2 天）→ 1 个
总计 1+1+0+1+1 = **4** 个。

【数据范围】

1 ≤ n ≤ 100，0 ≤ 步数 ≤ 50000。

【提示】

用一个变量 `cnt` 记录当前连续达标天数。每天判断：如果步数 ≥ 8000 则 `cnt++`，然后根据 `cnt` 的值决定当天的红包数——`cnt` 为 1~2 得 1 个，3~4 得 2 个，≥5 得 3 个，累加到答案；否则 `cnt = 0`。

### 第 2 题 跨类凑单抽奖 ★★ [P9733]（100 分）— 枚举 <a href="https://fslong.iok.la/problem/P9733" target="_blank">🧑‍💻 在线答题</a>

【题目描述】

学校旁边的小卖部搞活动：买**两件不同类别**的商品，只要两件商品的价格之和恰好等于 20 元，就可以参加抽奖一次。

王老师记录了当天 n 件商品的信息，每件商品有价格 a[i] 和类别 c[i]（1 = 饮料，2 = 零食，3 = 文具）。"两件不同类别"是指这两件商品的类别编号不能相同（比如一杯饮料配一包零食可以，两杯饮料不行）。

请你帮忙统计一下，有多少种不同的购买方案？这里"不同方案"指选出两件不同的商品（不考虑先后顺序，即选第 i 件和第 j 件与选第 j 件和第 i 件算同一种）。

【输入格式】

第一行，一个整数 n（2 ≤ n ≤ 1000）。

第二行，n 个正整数 a[i]，表示每件商品的价格（1 ≤ a[i] ≤ 100）。

第三行，n 个整数 c[i]，表示每件商品的类别（1 ≤ c[i] ≤ 3）。

【输出格式】

一行，一个整数，表示满足条件的方案数。

【样例 1 输入】

```
6
5 15 7 13 10 10
1 2 1 2 1 3
```

【样例 1 输出】

```
3
```

【样例 1 解释】

价格和为 20 的商品对有：(5,15) 类别 1、2 不同 ✓；(7,13) 类别 1、2 不同 ✓；(10,10) 指第 5 件（价 10、类 1）与第 6 件（价 10、类 3），类别不同 ✓。共 3 种。

【样例 2 输入】

```
4
10 10 10 10
1 2 1 2
```

【样例 2 输出】

```
4
```

【样例 2 解释】

四件商品都价 10，要凑 20 必须选两件都价 10。共 C(4,2)=6 对，但同类不能组队：第 1、3 件同属类别 1 ✗，第 2、4 件同属类别 2 ✗；其余 (1,2)、(1,4)、(3,2)、(3,4) 均为异类 ✓，共 4 种。注意：相同价格的商品因编号不同算不同商品，组合数要算对。

【样例 3 输入】

```
3
5 6 7
1 2 3
```

【样例 3 输出】

```
0
```

【样例 3 解释】

任意两件价格之和都不等于 20，方案数为 0。

【数据范围】

2 ≤ n ≤ 1000，1 ≤ a[i] ≤ 100，1 ≤ c[i] ≤ 3。

【提示】

两层循环枚举所有 i < j 的无序对。判断条件有两个必须**同时满足**：① 类别不同 `c[i] != c[j]`；② 价格之和恰好 20 `a[i] + a[j] == 20`。用 i < j 天然保证不重复计数、且不会把同一件商品选两次。

### 第 3 题 进步排行榜 ★★ [P9734]（100 分）— 结构体排序 <a href="https://fslong.iok.la/problem/P9734" target="_blank">🧑‍💻 在线答题</a>

【题目描述】

期末考试后，王老师要出一份"进步排行榜"。全班有 n 名同学，每人有：学号 id、上次考试总分 old、本次考试语文 ch、数学 ma、英语 en。

先算出两个量：
- **本次总分** s = ch + ma + en
- **进步分** p = s − old（可以为负，表示退步了）

排名规则：
1. 普通生按本次总分 s 从高到低排；
2. 若 s 相同，进步分 p 高者排前面；
3. 若 s 和 p 都相同，学号 id 小者排前面。

但是，如果某同学**本次考试任一单科低于 60 分**，则该同学标记为"待定"，必须排在所有普通生**之后**；待定生之间仍按上面的 1~3 条规则排序。

请你按最终排名，每行输出一个学号。

【输入格式】

第一行，一个整数 n（1 ≤ n ≤ 1000）。

接下来 n 行，每行五个整数：学号 id、上次总分 old、语文 ch、数学 ma、英语 en（1 ≤ id ≤ n，0 ≤ old, ch, ma, en ≤ 300）。

【输出格式】

输出 n 行，每行一个整数，按排名从高到低输出学号。

【样例 1 输入】

```
3
1 200 90 90 90
2 260 85 95 85
3 100 95 80 85
```

【样例 1 输出】

```
1
2
3
```

【样例 1 解释】

- 1 号：s = 270，p = 70，无不及格 → 普通
- 2 号：s = 265，p = 5，无不及格 → 普通
- 3 号：s = 260，p = 160，无不及格 → 普通

按 s 降序：1(270) > 2(265) > 3(260)，输出 1 2 3。

【样例 2 输入】

```
4
1 200 50 90 90
2 100 90 95 85
3 200 90 90 90
4 260 80 80 80
```

【样例 2 输出】

```
2
3
4
1
```

【样例 2 解释】

- 1 号：语文 50 < 60 → 待定；s = 230，p = 30
- 2 号：s = 270，p = 170 → 普通
- 3 号：s = 270，p = 70 → 普通
- 4 号：s = 240，p = −20 → 普通

普通生按 s 排：2、3 同 s=270，比 p：170 > 70 → 2 在前、3 在后；再 4(s=240)。待定生 1 号排最后。最终：2 3 4 1。

【数据范围】

1 ≤ n ≤ 1000，1 ≤ id ≤ n，0 ≤ old, ch, ma, en ≤ 300。

【提示】

先为每人算出 s 和 p，并判断是否有单科 < 60 得到 `pend` 标记（0 普通 / 1 待定）。排序的 cmp 第一关键字是 `pend` 升序（待定排后面），之后才是 s 降序、p 降序、id 升序。注意 s 和 p 都是要先计算的**派生量**，不能直接拿原始输入排。

### 第 4 题 最大 3×3 子矩阵 ★ [P9735]（100 分）— 二维数组 <a href="https://fslong.iok.la/problem/P9735" target="_blank">🧑‍💻 在线答题</a>

【题目描述】

冯老师有一个 n 行 m 列的数字矩阵，每个格子里有一个整数（可能为负）。她想从中找一个 **3 行 3 列**（即 3×3）的连续子矩阵，使得这个子矩阵内所有数的和最大。请你帮她算一算，这个最大的和是多少？

【输入格式】

第一行，两个整数 n 和 m（3 ≤ n, m ≤ 50）。

接下来 n 行，每行 m 个整数，表示矩阵中的数字（−100 ≤ 数字 ≤ 100）。

【输出格式】

一行，一个整数，表示最大的 3×3 子矩阵的和。

【样例 1 输入】

```
3 4
10 1 1 1
10 1 1 1
10 1 1 1
```

【样例 1 输出】

```
33
```

【样例 1 解释】

行数固定为 3，可取的 3 列连续区域有 2 个：
- 列 1~3：10+1+1+10+1+1+10+1+1 = **33**
- 列 2~4：1+1+1+1+1+1+1+1+1 = **9**
最大值为 **33**。

【样例 2 输入】

```
4 4
-5 -5 -5 -5
-5 10 10 -5
-5 10 10 -5
-5 -5 -5 -5
```

【样例 2 输出】

```
15
```

【样例 2 解释】

所有可能的 3×3 子矩阵：
- 左上角 (1,1)：全是 −5，和 = −45
- 左上角 (1,2)：中间 4 个 10 加上下左右 −5，和 = 15
- 左上角 (2,1)：同上，和 = 15
- 左上角 (2,2)：中间 4 个 10 加上下左右 −5，和 = 15
最大值为 **15**。（注意：矩阵含负数，答案的初始值不能设 0，要设一个很小的数。）

【数据范围】

3 ≤ n, m ≤ 50，−100 ≤ 矩阵中的数字 ≤ 100。

【提示】

枚举所有 3×3 子矩阵的左上角 (i, j)，i 从 1 到 n−2，j 从 1 到 m−2。对每个左上角，计算 9 个数的和并更新最大值。初始化答案为一个很小的数（如 −10⁹），因为和可能为负。
