### 第 16 题 <a href="https://fslong.iok.la/problem/P9432" target="_blank">跑步配速计算器（100 分）</a>

**题目描述**  
小明跑了10公里，用时t分钟t_秒。计算平均配速（每公里所需时间，格式为"X分Y秒"）。

**输入格式**  
一行，两个整数，分别是分钟数和秒数（如 45 30 表示45分30秒跑完10公里）。

**输出格式**  
两行，第一行为配速的分钟数，第二行为配速的秒数。

**计算方式**
- 总秒数 = 分钟 × 60 + 秒
- 每公里秒数 = 总秒数 ÷ 10
- 再转换回分钟和秒

**样例输入**
```
45 30
```

**样例输出**
```
4
33
```

**提示**  
45分30秒 = 2730秒，除以10得273秒/公里，273÷60=4分…33秒。

### 第 18 题 <a href="https://fslong.iok.la/problem/P9434" target="_blank">镜子文字（100 分）</a>

**题目描述**

小华在镜子前写单词，他发现了一件神奇的事：

有些单词在镜子里看，从左往右读，和纸上写的**完全一样**！
比如 `"abcba"`，纸上写的是 a-b-c-b-a，镜子里看到的也是 a-b-c-b-a。

但有些单词就不行。比如 `"hello"`，纸上写的是 h-e-l-l-o，镜子里看到的却是 o-l-l-e-h——完全反了！

小华把这种"镜子里和纸上看起来一样"的单词叫做**对称词**🪞。

输入一个单词，判断它是不是对称词。

**输入格式**
- 一行，一个字符串 s（1 ≤ |s| ≤ 100，只包含小写字母）

**输出格式**
- 如果是输出 `Yes`，否则输出 `No`

**样例输入**

```
abcba
```

**样例输出**

```
Yes
```

**样例输入2**

```
hello
```

**样例输出2**

```
No
```

**提示**
- 镜子里看到的单词，其实就是把原单词**倒过来**写
- 把原单词倒过来，和原单词比一比：一样就是对称词，不一样就不是
- 可以用双指针从两端往中间比，也可以直接 reverse 后比较

### 第 19 题 <a href="https://fslong.iok.la/problem/P9435" target="_blank">指针找中位数（100 分）</a>

**题目描述**  
输入 n（奇数，1 ≤ n ≤ 99）和 n 个整数，用指针找出中位数（排序后中间那个数）。

**要求**  
1. 必须用指针实现选择排序（不能用下标 `[]`）
2. 找到中位数后输出

**输入格式**  
第一行一个正整数 n。  
第二行 n 个整数。

**输出格式**  
一行，一个整数，表示中位数。

**样例输入**
```
5
3 1 4 1 5
```

**样例输出**
```
3
```

**提示**  
用 `*(arr + i)` 访问数组元素。排序后第 n/2 个元素（下标从0开始）即为中位数。

### 第 20 题 <a href="https://fslong.iok.la/problem/P9436" target="_blank">指针数组统计（100 分）</a>

**题目描述**  
输入 n（1 ≤ n ≤ 100）和 n 个整数，统计正数、负数、零的个数。

**要求**  
1. 必须用指针遍历数组，不能使用下标 `[]`。
2. 编写函数 `void count(int *arr, int n, int *pos, int *neg, int *zero)`，用指针参数返回结果。

**输入格式**  
第一行一个整数 n。  
第二行 n 个整数。

**输出格式**  
三行，依次为正数个数、负数个数、零的个数。

**样例输入**
```
6
-2 0 3 5 -1 7
```

**样例输出**
```
3
2
1
```

**提示**  
函数内用指针遍历数组，根据 `*p` 的正负零分类，通过指针参数修改对应计数变量。
