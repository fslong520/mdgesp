### 第 1 题 快递单号校验 ★ [P9742]（100 分）— 字符串模拟 <a href="https://fslong.iok.la/problem/P9742" target="_blank">🧑‍💻 在线答题</a>

【题目描述】

双十一到了，冯老师帮快递站分拣包裹。快递站的系统会对每个单号自动校验，只有通过全部三条规则的单号才能入库扫描，否则就是"问题件"要人工处理。冯老师想写个程序快速筛出合格的单号。

单号是一个由字母（A-Z、a-z）和数字（0-9）组成的字符串，长度恰好为 10。校验规则如下：

1. **无相邻重复**：不能有两个相同的相邻字符（如 "AA"、"55" 不允许，但间隔出现的相同字符可以，如 "A1A"）
2. **字母数为偶数**：单号中所有字母的总个数必须是偶数
3. **数字数为偶数**：单号中所有数字的总个数必须是偶数

给定 n 个快递单号，请统计其中有多少个通过了全部校验。

【输入格式】

第一行，一个整数 n（1 ≤ n ≤ 1000）。

接下来 n 行，每行一个长度恰好为 10 的字符串，表示一个快递单号。

【输出格式】

一行，一个整数，表示通过校验的单号数量。

【样例 1 输入】

```
4
a1b2c3d4e5
AABB112233
Ab1Cd2Ef3Gh
Aa1B2c3D4e
```

【样例 1 输出】

```
1
```

【样例 1 解释】

- "a1b2c3d4e5"：无相邻重复 ✓，字母 5 个（奇数）✗
- "AABB112233"：有 "AA" 相邻重复 ✗
- "Ab1Cd2Ef3Gh"：无相邻重复 ✓，字母 6 个（偶数）✓，数字 3 个（奇数）✗
- "Aa1B2c3D4e"：无相邻重复 ✓，字母 6 个（偶数）✓，数字 4 个（偶数）✓

只有最后一个通过全部校验，输出 1。

【样例 2 输入】

```
3
aB3cD5eF7g
1234567890
AbCdEfGhIj
```

【样例 2 输出】

```
2
```

【样例 2 解释】

- "aB3cD5eF7g"：无相邻重复 ✓，字母 6 个（偶数）✓，数字 3 个（奇数）✗
- "1234567890"：无相邻重复 ✓，字母 0 个（偶数）✓，数字 10 个（偶数）✓
- "AbCdEfGhIj"：无相邻重复 ✓，字母 10 个（偶数）✓，数字 0 个（偶数）✓

后两个通过校验，输出 2。

【数据范围】

1 ≤ n ≤ 1000，每个单号长度恰好为 10，仅由大小写字母和数字组成。

【提示】

对每个单号做三件事：① 从头到尾扫描一遍，检查 `s[i] == s[i-1]`；② 遍历统计字母个数和数字个数；③ 判断两个计数是否均为偶数。三条都满足才算通过。时间复杂度 O(10n)，轻松通过。

### 第 2 题 关卡挑战配对 ★★ [P9743]（100 分）— 枚举 <a href="https://fslong.iok.la/problem/P9743" target="_blank">🧑‍💻 在线答题</a>

【题目描述】

学校旁边的电玩城新出了一款闯关游戏，共有 n 个关卡。每个关卡 i 有一个难度值 d[i]，通关后能拿到的奖励代币为 r[i]。

电玩城有个"搭档模式"：选两个不同的关卡同时挑战，通关后奖励叠加。但不是任意两个关卡都能搭档——只有当两个关卡的难度差**不超过** k（即 |d[i] − d[j]| ≤ k）时，机器才会允许你搭档闯关。

小明想知道，在所有合法的搭档组合中，最多能拿到多少奖励代币？

如果没有任何合法的搭档组合，输出 −1。

【输入格式】

第一行，两个整数 n 和 k（2 ≤ n ≤ 5000，1 ≤ k ≤ 1000）。

接下来 n 行，每行两个整数 d[i] 和 r[i]（1 ≤ d[i] ≤ 1000，1 ≤ r[i] ≤ 1000），分别表示第 i 个关卡的难度和奖励代币数。

【输出格式】

一行，一个整数，表示最大的奖励代币之和；若无合法搭档，输出 −1。

【样例 1 输入】

```
4 2
3 15
5 25
6 10
8 20
```

【样例 1 输出】

```
40
```

【样例 1 解释】

四个关卡：(3,15)、(5,25)、(6,10)、(8,20)。

合法搭档（难度差 ≤ 2）及其奖励和：

| 搭档 | 难度差 | 奖励和 |
|------|--------|--------|
| (3,15) 与 (5,25) | 2 | 40 |
| (5,25) 与 (6,10) | 1 | 35 |
| (6,10) 与 (8,20) | 2 | 30 |

最大奖励和为 **40**。

【样例 2 输入】

```
6 5
1 100
3 80
6 90
10 70
12 60
15 50
```

【样例 2 输出】

```
190
```

【样例 2 解释】

合法搭档中奖励和最大的是 (1,100) 与 (6,90)，难度差 5，奖励和 190。

【样例 3 输入】

```
3 0
10 100
11 50
12 80
```

【样例 3 输出】

```
-1
```

【样例 3 解释】

k=0 要求难度完全相同。三个关卡难度各不相同（10、11、12），不存在合法搭档，输出 −1。

【数据范围】

2 ≤ n ≤ 5000，1 ≤ d[i] ≤ 1000，1 ≤ r[i] ≤ 1000，1 ≤ k ≤ 1000。

【提示】

两重循环枚举所有 i < j 的关卡对。对每一对，先判断 |d[i] − d[j]| ≤ k 是否成立，成立则用 r[i] + r[j] 更新答案最大值。初始化答案为 −1，若全程没有更新过则输出 −1。时间复杂度 O(n²)，n = 5000 时约 1250 万次运算，可在 1 秒内完成。

### 第 3 题 冷区探测 ★ [P9744]（100 分）— 二维矩阵枚举 <a href="https://fslong.iok.la/problem/P9744" target="_blank">🧑‍💻 在线答题</a>

【题目描述】

冯老师在玩一款吃鸡手游，游戏地图被分成 n 行 m 列的网格，每个格子有一个"危险值"（数值越大代表该区域敌人越多，越容易刚枪；负数表示安全区）。

冯老师有一个 **2 行 k 列** 的扫描框，可以框住地图上任意一个连续的 2×k 区域。框内所有格子的危险值之和就是该区域的"危险得分"。

为了安稳发育，冯老师想找到危险得分**最小**的 2×k 区域跳伞。请你帮她算一算，这个最小危险得分是多少？

【输入格式】

第一行，三个整数 n、m 和 k（2 ≤ n ≤ 50，2 ≤ k ≤ m ≤ 50）。

接下来 n 行，每行 m 个整数，表示地图各格子的危险值（−100 ≤ 危险值 ≤ 100）。

【输出格式】

一行，一个整数，表示最小的危险得分。

【样例 1 输入】

```
2 5 3
5 -10  3  -8  2
2  -7  1  -4  6
```

【样例 1 输出】

```
-25
```

【样例 1 解释】

地图只有 2 行，k=3，可取的 2×3 区域有 3 个：

- 左上角 (1,1)：5 + (−10) + 3 + 2 + (−7) + 1 = **−6**
- 左上角 (1,2)：(−10) + 3 + (−8) + (−7) + 1 + (−4) = **−25**
- 左上角 (1,3)：3 + (−8) + 2 + 1 + (−4) + 6 = **0**

最小值为 **−25**。

【样例 2 输入】

```
3 6 4
10  5  8  3  12  7
-2  7  4 -6   9  1
 3 -1  6  2  -5  4
```

【样例 2 输出】

```
13
```

【样例 2 解释】

n=3, m=6, k=4，可取的 2×4 区域有 6 个：

| 左上角 | 危险得分 |
|--------|----------|
| (1,1) | 10+5+8+3+(−2)+7+4+(−6) = 29 |
| (1,2) | 5+8+3+12+7+4+(−6)+9 = 42 |
| (1,3) | 8+3+12+7+(−6)+9+1+2 = 36 |
| (2,1) | (−2)+7+4+(−6)+3+(−1)+6+2 = 13 |
| (2,2) | 7+4+(−6)+9+(−1)+6+2+(−5) = 16 |
| (2,3) | 4+(−6)+9+1+6+2+(−5)+4 = 15 |

最小值为 **13**。

【数据范围】

2 ≤ n ≤ 50，2 ≤ k ≤ m ≤ 50，−100 ≤ 危险值 ≤ 100。

【提示】

枚举所有 2×k 子矩阵的左上角 (i, j)：i 从 0 到 n−2，j 从 0 到 m−k。对每个左上角，用两层循环累加 k 列×2 行共 2k 个格子的和，更新最小值。答案初始化为一个很大的正数（如 10⁹），因为所有和都可能小于它。时间复杂度 O(n × (m−k+1) × k)。

### 第 4 题 运动会分组排名 ★★ [P9745]（100 分）— 结构体排序 <a href="https://fslong.iok.la/problem/P9745" target="_blank">🧑‍💻 在线答题</a>

【题目描述】

学校运动会，n 个同学报名参赛。每人有昵称、组别和一个成绩值。

组别有三种：

- **短跑组**（group = 1）：成绩是用时（秒，**越小越好**）
- **长跑组**（group = 2）：成绩是用时（秒，**越小越好**）
- **田赛组**（group = 3）：成绩是距离（米，**越大越好**）

注意：短跑组和长跑组按成绩**从小到大**排（用时少的厉害），田赛组按成绩**从大到小**排（投得远的厉害）。

排名时，先按组别顺序输出：**短跑组 → 长跑组 → 田赛组**。每个组内部按各自规则排好序后，依次输出。如果组内成绩相同，按昵称的字典序从小到大排。

请按最终顺序输出所有同学的昵称。

【输入格式】

第一行，一个整数 n（1 ≤ n ≤ 1000）。

接下来 n 行，每行三个数据：一个字符串昵称、一个整数 group、一个浮点数 score（1 ≤ group ≤ 3，1 ≤ score ≤ 10000，结果保留一位小数）。

【输出格式】

输出 n 行，每行一个昵称，按最终排名顺序输出。

【样例 1 输入】

```
6
Alice 1 12.5
Bob 3 8.2
Charlie 2 180.0
Diana 1 11.8
Eve 3 7.5
Frank 2 175.0
```

【样例 1 输出】

```
Diana
Alice
Frank
Charlie
Bob
Eve
```

【样例 1 解释】

分组：

- 短跑组(1)：Alice(12.5)、Diana(11.8) → 按用时从小到大：Diana(11.8) → Alice(12.5)
- 长跑组(2)：Charlie(180.0)、Frank(175.0) → 按用时从小到大：Frank(175.0) → Charlie(180.0)
- 田赛组(3)：Bob(8.2)、Eve(7.5) → 按距离从大到小：Bob(8.2) → Eve(7.5)

按"短跑→长跑→田赛"拼接输出：Diana, Alice, Frank, Charlie, Bob, Eve。

【样例 2 输入】

```
5
Amy 1 12.0
Anna 1 12.0
Ben 2 300.0
Bob 2 300.0
Cathy 3 6.0
```

【样例 2 输出】

```
Amy
Anna
Ben
Bob
Cathy
```

【样例 2 解释】

- 短跑组：Amy(12.0)、Anna(12.0) → 成绩相同，按昵称字典序：Amy → Anna
- 长跑组：Ben(300.0)、Bob(300.0) → 成绩相同，按昵称字典序：Ben → Bob
- 田赛组：Cathy(6.0) → 只有一人

输出：Amy, Anna, Ben, Bob, Cathy。

【数据范围】

1 ≤ n ≤ 1000，1 ≤ group ≤ 3，1 ≤ score ≤ 10000，score 保留一位小数。昵称仅含大小写字母，长度不超过 20。

【提示】

定义结构体存储昵称 name、组别 group、成绩 score。读入所有数据后，用**三次 sort** 分别对短跑组、长跑组、田赛组排序，或者用一个 cmp 函数统一处理：cmp 中先判断两人的 group 是否相同，不同组的按 group 编号排（1 < 2 < 3）；同组时，短跑和长跑用 `a.score < b.score`（升序），田赛用 `a.score > b.score`（降序）；成绩相同时按 name 升序。排序后直接输出 name 即可。
