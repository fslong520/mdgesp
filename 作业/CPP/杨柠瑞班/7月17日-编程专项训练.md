### 第 1 题 斗图统计 ★ [P9751]（100 分）— 字符串模拟 <a href="https://fslong.iok.la/problem/P9751" target="_blank">🧑‍💻 在线答题</a>

【题目描述】

暑假到了，冯老师在家族群里抢红包抢得不亦乐乎。

抢完红包，大家开始"斗图"——发各种表情包。但二叔的手机太旧了，表情包刷不出来，全是方框框。

"你们发的啥我看不到啊！"二叔急了。

表弟说："二叔，我教你发文本表情！用键盘就能打出来！"

于是家族群开启了文本表情模式：
- `:)` 表示微笑
- `:(` 表示难过
- `:D` 表示大笑
- `:P` 表示吐舌
- `;(` 表示哭泣
- `;p` 表示调皮

二叔学会了，开始疯狂刷屏：":)：D;(:D……"

冯老师想统计一下，群里 n 条消息中，一共出现了多少个文本表情。

注意：一条消息里可能有多个表情，比如 ":):(" 算 2 个。

【输入格式】

第一行，一个整数 n（1 ≤ n ≤ 1000）。

接下来 n 行，每行一个由 `:` `;` `(` `)` `D` `P` `p` 组成的字符串（长度不超过 100），表示一条消息。

【输出格式】

一行，一个整数，表示文本表情的总个数。

【样例 1 输入】

```
2
:):(:D
:P;(
```

【样例 1 输出】

```
5
```

【样例 1 解释】

- 第一条消息 ":):(:D"：表情依次为 ":)" 、":(" 、":D"，共 3 个
- 第二条消息 ":P;("：表情依次为 ":P" 、";("，共 2 个

总共 3 + 2 = 5 个。

【样例 2 输入】

```
2
;(
:):(;p:D
```

【样例 2 输出】

```
5
```

【样例 2 解释】

- 第一条消息 ";("：1 个表情
- 第二条消息 ":):(;p:D"：":)"、":("、";p"、":D" 共 4 个

总共 1 + 4 = 5 个。

【数据范围】

1 ≤ n ≤ 1000，每条消息长度 ≤ 100。

【提示】

遍历每条消息的每个字符，对于位置 i，判断 s[i] 和 s[i+1] 能不能组成一个文本表情。6 种表情的配对是固定的，写一个函数或 if 判断即可。注意循环到倒数第二个字符，不要越界。时间复杂度 O(100n)。

### 第 2 题 密室逃脱 ★★ [P9752]（100 分）— 计数 <a href="https://fslong.iok.la/problem/P9752" target="_blank">🧑‍💻 在线答题</a>

【题目描述】

暑假到了，冯老师带同学们去玩密室逃脱。

大家走进一个古堡密室，墙上嵌着一排数字按钮，屏幕上滚动着一行文字：

"欢迎来到数字迷宫。眼前这些数字，每一个都成双成对，只有一个孤零零地站在这里。找到它，门就会打开。"

同学们数了数，确实——除了一个数字，其他每个数字都恰好出现了偶数次（2 次、4 次、6 次……）。那个落单的数字只出现了奇数次（1 次、3 次、5 次……）。

找出它，才能逃出密室！

请你帮冯老师找出这个落单的数字。

【输入格式】

第一行，一个整数 n（1 ≤ n ≤ 1000，n 为奇数）。

第二行，n 个整数（1 ≤ 每个数 ≤ 1000），表示每张卡片上的数字。

【输出格式】

一行，一个整数，表示落单的数字。

【样例 1 输入】

```
7
3 1 5 3 5 1 7
```

【样例 1 输出】

```
7
```

【样例 1 解释】

7 个数字中，3 出现 2 次，1 出现 2 次，5 出现 2 次，都是成对的。只有 7 出现了 1 次（奇数次），是落单的数字。

【样例 2 输入】

```
5
100 50 100 50 99
```

【样例 2 输出】

```
99
```

【样例 2 解释】

100 和 50 各出现 2 次，99 出现 1 次，是落单的数字。

【数据范围】

1 ≤ n ≤ 1000（n 为奇数），1 ≤ 每个数 ≤ 1000。

【提示】

用一个长度为 1001 的计数数组 cnt，初始全 0。每读入一个数字 x，执行 cnt[x]++。全部读完后，遍历数组找到 cnt[i] 为奇数的 i，输出即可。核心思想：**用数组下标当作计数器**，统计每个数字出现的次数。时间复杂度 O(n + 1000)。

### 第 3 题 古镇模拟器 ★ [P9753]（100 分）— 二维数组 <a href="https://fslong.iok.la/problem/P9753" target="_blank">🧑‍💻 在线答题</a>

【题目描述】

暑假到了，冯老师躺在沙发上刷短视频。

一个游戏广告跳了出来——画面里是一座古色古香的小镇，小桥流水，青石板路，一个穿汉服的小姐姐对着镜头招手：

"镇长大人，快来建设你的古镇吧！"

广告词写着——"《古镇模拟器》——当镇长，建古镇，日进斗金！零基础也能当老板！"

冯老师心动了，手指不由自主地点了"下载"。

进度条：1%……35%……78%……100%！

游戏打开了！

一张古镇地图映入眼帘。冯老师仔细一看，地图被分成了 n 行 m 列的网格，每个格子里标注着不同的设施：

有的设施能赚钱——火锅店 +50 金币/天，奶茶店 +20 金币/天，纪念品店 +15 金币/天……

有的设施却要花钱维护——古井 −8 金币/天，祠堂 −12 金币/天，花坛 −5 金币/天……

也就是说，每个格子是一个整数，**正数代表每天赚的金币，负数代表每天亏的钱**。

游戏规则：冯老师可以买下地图上"任意一块连续的长方形区域"来经营。但她现在的金币有限，只能买一块。

冯老师想找一块总利润最大的区域。如果所有可能的区域都是亏钱的（最大利润 ≤ 0），她就卸载游戏，输出 0 表示不玩了。

请你帮冯老师算一算，她最多每天能赚多少金币？

【输入格式】

第一行，两个整数 n 和 m（1 ≤ n, m ≤ 30）。

接下来 n 行，每行 m 个整数（−100 ≤ 每个数 ≤ 100），表示每个格子的利润。

【输出格式】

一行，一个整数，表示最大矩形利润。如果最大利润小于等于 0，输出 0。

【样例 1 输入】

```
3 3
1 2 3
4 5 6
7 8 9
```

【样例 1 输出】

```
45
```

【样例 1 解释】

全部数字都是正数，整个 3×3 矩阵就是最大的矩形，和为 45。

【样例 2 输入】

```
2 3
-5 -3 -2
-1 -4 -6
```

【样例 2 输出】

```
0
```

【样例 2 解释】

所有数字都是负数，任何矩形都是亏钱的，输出 0。

【数据范围】

1 ≤ n, m ≤ 30，−100 ≤ 每个数 ≤ 100。

【提示】

先构建二维前缀和数组 s，其中 s[i][j] 表示从 (1,1) 到 (i,j) 的矩形之和。然后四重循环枚举所有矩形的左上角 (x1,y1) 和右下角 (x2,y2)，用公式 `sum = s[x2][y2] - s[x1-1][y2] - s[x2][y1-1] + s[x1-1][y1-1]` O(1) 计算矩形和，更新最大值。如果最终 max ≤ 0，输出 0。n,m ≤ 30 时 O(n²m²) 约 81 万次，非常快。

### 第 4 题 神同步 ★★ [P9754]（100 分）— 结构体排序 <a href="https://fslong.iok.la/problem/P9754" target="_blank">🧑‍💻 在线答题</a>

【题目描述】

暑假到了，小区举办"神同步"默契挑战赛！

这是一个考验两人默契的综艺闯关节目。每队挑战三个项目，每个项目由裁判打分（1-100 分）：

- 第 1 关：**你画我猜**——一个人画，一个人猜，猜对越多分越高
- 第 2 关：**双人跳绳**——两人一起跳，次数越多分越高
- 第 3 关：**蒙眼喂食**——一个人蒙眼，另一个人指挥，喂到嘴里才算

三关下来，每队有三个分数。排名规则很特别——节目组说：

> "先看亮点——你有一项特别牛的表现吗？亮点相同，看下限——最差的那一项也不能太难看。"

也就是说：
1. 先按**最高分**从高到低排
2. 最高分相同，按**最低分**从高到低排
3. 还相同，按**队名字典序从小到大**排

请你按最终排名输出所有队伍名称。

【输入格式】

第一行，一个整数 n（1 ≤ n ≤ 1000）。

接下来 n 行，每行格式：队名 第1关分 第2关分 第3关分。队名为仅含大小写字母的字符串（长度 ≤ 20），三关分数为 1-100 的整数。

【输出格式】

输出 n 行，每行一个队名，按排名顺序输出。

【样例 1 输入】

```
4
默契搭档 92 88 95
快乐双星 95 70 80
超能战队 80 90 85
友谊小船 70 75 85
```

【样例 1 输出】

```
默契搭档
快乐双星
超能战队
友谊小船
```

【样例 1 解释】

| 队名 | 三关分数 | 最高分 | 最低分 |
|------|---------|-------|-------|
| 默契搭档 | 92,88,95 | **95** | **88** |
| 快乐双星 | 95,70,80 | **95** | **70** |
| 超能战队 | 80,90,85 | **90** | **80** |
| 友谊小船 | 70,75,85 | **85** | **70** |

按最高分排：默契(95)=快乐(95) > 超能(90) > 友谊(85)。默契与快乐最高分相同，比最低分：默契(88) > 快乐(70)，所以默契在前。

【样例 2 输入】

```
3
Alpha 90 85 80
Beta 85 80 85
Gamma 70 75 80
```

【样例 2 输出】

```
Alpha
Beta
Gamma
```

【样例 2 解释】

- Alpha：最高 90，最低 80
- Beta：最高 85，最低 80
- Gamma：最高 80，最低 70

按最高分：Alpha(90) > Beta(85) > Gamma(80)，直接分出先后。

【数据范围】

1 ≤ n ≤ 1000，队名长度 ≤ 20，1 ≤ 每关分数 ≤ 100。

【提示】

定义结构体存储队名 name 和三个分数 a,b,c。读入后对每个队伍计算最高分 mx = max({a,b,c}) 和最低分 mn = min({a,b,c})。排序自定义 cmp：先比 mx 降序，再比 mn 降序，最后比 name 升序。排序后直接输出 name 即可。注意 max 和 min 函数可以嵌套使用，如 `max(a, max(b, c))`。时间复杂度 O(n log n)。
