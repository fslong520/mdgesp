### 第 1 题 运动会报名 ★ [P9769]（100 分）— 字符串模拟 <a href="https://fslong.iok.la/problem/P9769" target="_blank">🧑‍💻 在线答题</a>

【题目描述】

暑假到了，冯老师组织班级运动会。每班生成一个"报名码"——由大写字母（A-Z）和数字（0-9）组成的字符串。

但报名系统有个Bug：如果报名码里 **数字的总个数多于字母的总个数**，系统会报错无法提交。

请帮冯老师统计，n 个报名码中有多少个能成功提交？

【输入格式】

第一行，一个整数 n（1 ≤ n ≤ 1000）。

接下来 n 行，每行一个字符串，表示报名码。字符串仅含大写字母 A-Z 和数字 0-9，长度不超过 20。

【输出格式】

一行，一个整数，表示能成功提交的报名码个数（数字个数 ≤ 字母个数）。

【样例 1 输入】

```
3
ABC123
A1B2C3D4
12AB
```

【样例 1 输出】

```
3
```

【样例 1 解释】

| 报名码 | 字母数 | 数字数 | 结果 |
|--------|:-----:|:-----:|:----:|
| ABC123 | 3 | 3 | 3 ≤ 3 ✓ 通过 |
| A1B2C3D4 | 4 | 4 | 4 ≤ 4 ✓ 通过 |
| 12AB | 2 | 2 | 2 ≤ 2 ✓ 通过 |

全部 3 个都通过。

【样例 2 输入】

```
2
A123
ABCDEF12
```

【样例 2 输出】

```
1
```

【样例 2 解释】

- "A123"：字母 1 个（A），数字 3 个 → 3 > 1 ✗ 失败
- "ABCDEF12"：字母 6 个，数字 2 个 → 2 ≤ 6 ✓ 通过

只有第 2 个通过，输出 1。

【数据范围】

1 ≤ n ≤ 1000，每个字符串长度 ≤ 20。

【提示】

遍历每个字符，用 `isalpha(c)` 判断字母或用 `c >= 'A' && c <= 'Z'` 判断大写字母，`c >= '0' && c <= '9'` 判断数字。注意：本题字符串只有大写字母和数字，用 `c >= 'A'` 即可区分字母（'A'-'Z'）和数字（'0'-'9'）。统计字母数 letter 和数字数 digit，若 digit ≤ letter 则计数加一。时间复杂度 O(20n)。

### 第 2 题 方砖铺地 ★★ [P9770]（100 分）— 枚举 <a href="https://fslong.iok.la/problem/P9770" target="_blank">🧑‍💻 在线答题</a>

【题目描述】

暑假到了，冯老师要装修新家。他有一块 a 厘米 × b 厘米的矩形地面，想铺上正方形的瓷砖。

瓷砖的边长必须是整数厘米。为了美观，冯老师要求瓷砖边长**至少为 3 厘米**。

问有多少种不同规格的瓷砖（即不同的边长），可以恰好铺满整个地面（不切割瓷砖、不重叠）？

【输入格式】

一行，两个整数 a 和 b（1 ≤ a, b ≤ 1000）。

【输出格式】

一行，一个整数，表示符合条件的瓷砖边长种类数。如果没有符合条件的，输出 0。

【样例 1 输入】

```
12 8
```

【样例 1 输出】

```
1
```

【样例 1 解释】

12 和 8 的公因数有 1、2、4。其中 ≥ 3 的只有 4。所以只有边长 4 cm 的瓷砖，共 1 种。

【样例 2 输入】

```
30 15
```

【样例 2 输出】

```
3
```

【样例 2 解释】

30 和 15 的公因数有 1、3、5、15。其中 ≥ 3 的有 3、5、15，共 3 种。

【样例 3 输入】

```
5 7
```

【样例 3 输出】

```
0
```

【样例 3 解释】

5 和 7 互质，公因数只有 1。1 < 3，没有符合条件的边长，输出 0。

【数据范围】

1 ≤ a, b ≤ 1000。

【提示】

瓷砖边长必须能同时整除 a 和 b，即 `a % d == 0 && b % d == 0`。枚举可能的边长 d 从 3 到 min(a, b)，检查是否同时整除两者。计数符合条件的 d 即可。

### 第 3 题 寻找山峰 ★★ [P9771]（100 分）— 矩阵内枚举 <a href="https://fslong.iok.la/problem/P9771" target="_blank">🧑‍💻 在线答题</a>

【题目描述】

冯老师暑假去爬山了。登山队拿到一张 n 行 m 列的数字地图，每个格子里的数字表示该点的**海拔高度**（0-99）。

登山队想在地图上标记"山峰"——一个格子如果是山峰，它的海拔必须**严格高于上下左右四个相邻格子**。

注意：地图边界上的格子缺少相邻格，所以不可能是山峰。

请帮登山队统计地图上有多少座山峰。

【输入格式】

第一行，两个整数 n 和 m（3 ≤ n, m ≤ 50）。

接下来 n 行，每行 m 个整数（0-99），表示海拔高度。

【输出格式】

一行，一个整数，表示山峰的数量。

【样例 1 输入】

```
3 3
1 2 3
4 5 6
7 8 9
```

【样例 1 输出】

```
0
```

【样例 1 解释】

3×3 地图中，只有正中间 (2,2)=5 有完整的 4 个邻居（上=2、下=8、左=4、右=6）。5 < 6 且 5 < 8，不是山峰。输出 0。

【样例 2 输入】

```
3 3
1 2 3
4 9 6
7 8 5
```

【样例 2 输出】

```
1
```

【样例 2 解释】

中间 (2,2)=9，4 个邻居为上=2、下=8、左=4、右=6，9 比它们都大，是山峰。输出 1。

【样例 3 输入】

```
4 4
5 3 4 6
2 9 1 8
7 4 3 2
1 5 6 4
```

【样例 3 输出】

```
1
```

【样例 3 解释】

检查所有不在边界的格子（即第 2~3 行、第 2~3 列）：

| 位置 | 值 | 上下左右 | 结果 |
|:----:|:--:|:---------|:----:|
| (2,2) | 9 | ↑3 ←2 →1 ↓4 | **9 > 全部 ✓** |
| (2,3) | 1 | ↑4 ←9 →8 ↓3 | 1 < 9 ✗ |
| (3,2) | 4 | ↑9 ←7 →3 ↓5 | 4 < 9 ✗ |
| (3,3) | 3 | ↑1 ←4 →2 ↓6 | 3 < 6 ✗ |

只有 (2,2) 是山峰，输出 1。

【数据范围】

3 ≤ n, m ≤ 50，0 ≤ 高度 ≤ 99。

【提示】

只检查内部格子——即行号 i 从 1 到 n-2、列号 j 从 1 到 m-2（行列均从 0 开始编号，0 为第 1 行/列，n-1/m-1 为末行/列）。用方向数组简化 4 方向比较：

```
int dx[4] = {-1, 1, 0, 0};
int dy[4] = { 0, 0,-1, 1};
```

循环检查 4 个邻居是否都小于当前格子值。全满足则计数加一。

### 第 4 题 古诗大赛 ★★ [P9772]（100 分）— 结构体排序 <a href="https://fslong.iok.la/problem/P9772" target="_blank">🧑‍💻 在线答题</a>

【题目描述】

班级举办古诗大赛，每位同学参加三轮比赛：

- **第 1 轮·朗诵**（满分 100）
- **第 2 轮·即兴创作**（满分 100）—— 最难的一轮
- **第 3 轮·常识问答**（满分 100）

最终得分的计算公式是：

> **最终分 = 朗诵 × 3 + 即兴创作 × 4 + 常识问答 × 3**

其中即兴创作的权重最高（×4），因为它最能体现古诗功底。

排名规则：
1. 最终分从高到低排
2. 最终分相同，按姓名拼音字典序从小到大排

请输出所有同学按排名顺序的姓名。

【输入格式】

第一行，一个整数 n（1 ≤ n ≤ 1000）。

接下来 n 行，每行格式：`姓名 朗诵分 即兴分 常识分`。

姓名仅含大小写字母（长度 ≤ 20），三部分数均为 0-100 的整数。

【输出格式】

输出 n 行，每行一个姓名，按排名从高到低输出。

【样例 1 输入】

```
4
Libai 85 90 92
Dufu 88 87 95
Wangwei 90 90 90
Baijuyi 95 80 85
```

【样例 1 输出】

```
Wangwei
Dufu
Libai
Baijuyi
```

【样例 1 解释】

| 姓名 | 朗诵×3 | 即兴×4 | 常识×3 | 最终分 | 排名 |
|------|:------:|:------:|:------:|:-----:|:----:|
| Wangwei | 90×3=270 | 90×4=360 | 90×3=270 | **900** | 1 |
| Dufu | 88×3=264 | 87×4=348 | 95×3=285 | **897** | 2 |
| Libai | 85×3=255 | 90×4=360 | 92×3=276 | **891** | 3 |
| Baijuyi | 95×3=285 | 80×4=320 | 85×3=255 | **860** | 4 |

最终分：Wangwei(900) > Dufu(897) > Libai(891) > Baijuyi(860)。

【样例 2 输入】

```
3
Sushi 80 80 80
Luyou 80 80 80
Liqingzhao 80 80 80
```

【样例 2 输出】

```
Liqingzhao
Luyou
Sushi
```

【样例 2 解释】

三人最终分相同：80×3+80×4+80×3=800。按字典序排：Liqingzhao < Luyou < Sushi。

【数据范围】

1 ≤ n ≤ 1000，姓名长度 ≤ 20，三部分数 0-100。

【提示】

定义结构体存储姓名 name 和三个分数。读入后计算最终分 `score = a*3 + b*4 + c*3`。自定义 cmp：先比 score 降序，再比 name 升序。排序后输出 name。权重计算用整数乘法，最终分最大 1000，int 足够。
