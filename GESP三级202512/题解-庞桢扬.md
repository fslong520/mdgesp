## 第 2 题【一维数组】(2分) ✗

以下关于标准 C++ 一维数组的描述，**错误的是**（ ）。

- A. 数组的下标从0开始
- B. 数组在内存中是连续存放的
- **C. 数组的长度只能在运行时才能确定** ← **正确答案**
- D. 可以在程序中修改数组某个元素的值

### 选项分析

| 选项 | 正误 | 理由 |
|:----:|:----:|:------|
| A | ✅ | C/C++ 数组下标从 0 开始，语法规定 |
| B | ✅ | 数组元素按声明顺序连续排列，这是下标随机访问的基础 |
| **C** | **❌** | **标准 C++ 中数组长度必须是编译期常量表达式，不可以在运行时确定。** `int n; cin >> n; int arr[n];` 在标准 C++ 中非法，依赖 VLA 扩展不可移植 |
| D | ✅ | 数组元素本质是变量，非 const 即可修改 |

### 小结
数组三特性：**下标从0、连续存放、编译期定长**。"运行时确定长度" 正是对标准 C++ 的错误描述。

## 第 11 题【自增自减】(2分) ✗

整型变量 `x` 的初始值为 10，以下代码的输出结果是（ ）。

```cpp
int t = x--;
t -= x;
cout << t << endl;
```

- A. 0
- B. -1
- **C. 1** ← **正确答案**
- D. 死循环

### 选项分析

逐行推演：

```
int x = 10;
int t = x--;   // 后置自减：t 得 x 原值 10，然后 x 减为 9
t -= x;        // t = 10 - 9 = 1
cout << t;     // 输出 1
```

| 表达式 | 结果 | 说明 |
|:-------|:----:|:-----|
| `x--` | 10 | 后置：先取原值，后自减 |
| `--x` | 9 | 前置：先自减，后取新值 |

后置 `x--` 与前置 `--x` 的区别是本题核心：**后置先使用再减，前置先减再使用**。`t -= x` 展开为 `t = t - x`，不是 `t = t-- - x`。

## 第 13 题【switch分支】(2分) ✗

以下代码运行后，`sum` 的结果是（ ）。

```cpp
int arr[5] = {2, 4, 6, 8, 10};
int sum = 0;
for (int i = 0; i < 5; i++) {
    switch (arr[i] % (1|2)) {
        case 0: sum += 1; break;
        case 1: sum += 2; break;
        case 2: sum += 3; break;
    }
}
```

- A. 10
- **B. 11** ← **正确答案**
- C. 14
- D. 15

### 选项分析

**第一步**：`1|2` 是**按位或**运算。

```
  00000001  (1)
| 00000010  (2)
-----------
  00000011  (3)
```

故 `arr[i] % (1|2)` 等价于 `arr[i] % 3`。注意不是逻辑或 `||`。

**第二步**：逐元素计算余数，分支累加。

| i | arr[i] | %3 | 命中的 case | sum |
|:-:|:------:|:--:|:----------:|:---:|
| 0 | 2 | 2 | case 2: sum += 3 | 3 |
| 1 | 4 | 1 | case 1: sum += 2 | 5 |
| 2 | 6 | 0 | case 0: sum += 1 | 6 |
| 3 | 8 | 2 | case 2: sum += 3 | 9 |
| 4 | 10 | 1 | case 1: sum += 2 | **11** |

每个 case 有 `break`，无穿透。最终 `sum = 11`。

### 关键点
- `1|2` 是位运算，值 3，不是逻辑或
- `%3` 余数只可能是 0、1、2，三个 case 全覆盖

## 第 14 题【abs函数】(2分) ✗

以下关于 C++ 中 `abs()` 函数的描述，**正确的是**（ ）。

- **A. `abs()` 函数可用于计算 int 类型整数的绝对值，头文件为 `<cmath>` 或 `<cstdlib>`** ← **正确答案**
- B. 调用 `abs(3.14)` 可以四舍五入得到 3
- C. `abs(-1, -2)` 的返回值是 1
- D. 若传入负数浮点数，`abs()` 会自动截断小数部分后返回整数绝对值

### 选项分析

| 选项 | 正误 | 理由 |
|:----:|:----:|:------|
| **A** | **✅** | `abs(int)` 由 `<cstdlib>` 提供；`<cmath>` 提供 `abs(double)` 等重载，也可处理 int（隐式提升）。两种头文件均可 |
| B | ❌ | `abs()` 计算**绝对值**，与四舍五入无关。`abs(3.14)` 若匹配 `abs(int)` 则截断为 3，若匹配 `abs(double)` 则返回 3.14。无论如何都不是"四舍五入" |
| C | ❌ | `abs()` 只接受**一个参数**，`abs(-1, -2)` 编译错误 |
| D | ❌ | `abs()` 本身没有"自动截断"行为。若匹配 `abs(int)` 则隐式类型转换时发生截断，但这并非 `abs()` 的功能；若匹配 `abs(double)` 则返回浮点数，不截断 |

### 要点
- 整数绝对值 → `abs()`，头文件 `<cstdlib>` 或 `<cmath>`
- 浮点绝对值 → `fabs()`，头文件 `<cmath>`
- `abs()` 只有一个参数

## 第 2 题（判断题 第 17 题）(2分) ✗

整型变量 a 的值为 3，浮点数变量 b 的值为 3.5，在 C++ 编译环境下，经过 `a *= b` 计算后，a 的值变为 10.5，此时如果输出 a，将输出 10.5。（ ）

**正确答案：F**

### 分析

`a *= b` 等价于 `a = a * b`。

```
a = 3       (int)
b = 3.5     (double)
a * b = 10.5  (double)
a = 10.5    → 赋值给 int 型，隐式转换，截断小数 → a = 10
```

两点错误：
1. a 的值是 **10**，不是 10.5
2. 输出 a 显示 **10**，不是 10.5

**规则**：整型变量接收浮点数时**截断小数部分**（直接舍弃，不四舍五入）。

## 第 9 题（判断题 第 24 题）(2分) ✗

给定一个正整数 a，当需要计算 -a 的补码时，有这样一个计算技巧：将 a 的二进制形式从右往左扫描，遇到第一个 1 之后，将找到的第一个 1 左边的所有位都取反，能得到 -a 的补码。（ ）

**正确答案：T**

### 分析

该技巧称为**补码快捷算法**，是正确的。验证：

例：求 -6 的补码（8位）

```
 6 的二进制:  0000 0110
 从右往左扫：第0位=0，第1位=1 → 遇到第一个1
 该1左边的所有位取反：0000 0 → 1111 1
 保持第1位和第0位不变
 结果:        1111 1010
```

验证：`1111 1010` → 取反 `1000 0101` → +1 → `1000 0110` = -6 ✅

**注意**：该技巧与"取反加一"等价。找到的第一个 1 及其右侧保持不变，左侧全部取反。

## 第 10 题（判断题 第 25 题）(2分) ✗

以下代码中，Hello 将被输出5次。

```cpp
for (int i = 0; i < 5; i++);
{
    cout << "Hello" << endl;
}
```

**正确答案：F**

### 分析

关键在**第1行末尾的分号**：

```cpp
for (int i = 0; i < 5; i++);  // ← 分号！循环体为空语句
{                              // ← 独立语句块，与 for 无关
    cout << "Hello" << endl;   // 只执行 1 次
}
```

- `for (int i = 0; i < 5; i++);` — 分号结束 for 语句，循环体是空语句，空转 5 次
- `{ cout << "Hello"; }` — 这是一个独立的复合语句块，**仅执行 1 次**

### 对比

```
有分号：for (...); { cout << "Hello"; } → 1 次
无分号：for (...){ cout << "Hello"; }   → 5 次
```
