# GESP 五级 2025年09月 真题 解析

---

## 一、选择题

**第 1 题** B — 链表擅长频繁中间/开头插入删除操作（已知位置 O(1)），数组则需移动大量元素 O(n)。A 错：数据量固定且读多写少适合数组（连续内存、缓存友好）；C 错：随机访问是数组优势，链表需遍历 O(n)；D 错：链表内存不必连续，数组才要求连续。

**第 2 题** C — 单链表删除结点需将前驱的 next 跳过待删结点。`del = cur->next` 记录待删结点，`cur->next = del->next` 将前驱 next 指向后继，跳过 del，最后 `delete del`。A 错：del 指向 cur 而非待删结点；B 错：`cur->next = del` 使前驱 next 指向自身，形成环；D 错：先 delete 再改指针会导致悬垂指针。

**第 3 题** A — Floyd 判环算法中，slow 每次走一步，fast 每次走两步。横线处应为 `slow = slow->next; fast = fast->next->next;`。B 错：slow 错误地指向 fast 的后继；C 错：fast 基于 slow 而非 fast 自身移动；D 错：slow 指向 fast->next，破坏了 slow 每次一步的要求。

**第 4 题** B — 因子成对出现：若 i 是因子，则 n/i 也是因子。枚举到 √n 即可避免重复。`i * i <= n` 等价于 i ≤ √n，为正确答案。A 错：枚举到 n 时间复杂度 O(n) 过高；C 错：n/2 仍为 O(n)；D 错：枚举到 n 同样效率低下。

**第 5 题** B — 辗转相除法中，当 b == 0 时，a 即为最大公约数。循环结束时 b == 0，a 保留了最终的 gcd，故返回 a。A 错：b 此时为 0；C 错：temp 为循环内局部变量，最后无意义；D 错：a * b 非 gcd。

**第 6 题** D — 埃氏筛从 i² 开始标记 i 的倍数，因小于 i² 的 i 的倍数（如 2i, 3i, …, (i−1)i）均含有小于 i 的质因子，已被更小的质数筛过。从 i² 开始可避免重复标记。A、B、C 均会重复标记已筛过的合数，降低效率。

**第 7 题** A — 线性筛（欧拉筛）的核心在于 `if (i % p == 0) break;`。当 i 能被 p 整除时，p 是 i 的最小质因子，若继续筛 i × (更大的质数)，所得合数的最小质因子将是 p 而非那个更大的质数，导致重复标记。B 错：条件反向；C 错：`i == p` 无法在合适时机 break；D 错：`i * p == n` 无意义。

**第 8 题** B — 选项 B 说法错误。线性筛理论复杂度 O(n) 优于埃氏筛 O(n log log n)，但"线性筛速度往往优于埃氏筛"不成立。对于 10⁶ 以内常见范围，埃氏筛实现简单、常数极小，实际运行速度往往快于线性筛。选项 D 正确描述了这一现象。

**第 9 题** B — 唯一分解定理（算术基本定理）：每个大于 1 的整数都能唯一地分解为素数幂的乘积（不计顺序）。A 错："任意素数"说法不精确；C 错：合数正可分解；D 错：那是素数的定义，非唯一分解定理。

**第 10 题** C — 二分答案标准模板：`countLE(mid) >= k` 表示第 k 小元素 ≤ mid，故收缩右边界 `hi = mid`；否则 mid 太小，收缩左边界 `lo = mid + 1`。最终 lo 为所求。A 错：`hi = mid - 1` 可能导致漏掉正确答案；B 错：`lo = mid` 在非终止条件下可能死循环；D 错：左右边界收缩方向错误。

**第 11 题** D — partition 中左右扫描顺序不可交换。本题 pivot 取 arr[low]（最左元素），必须先执行右侧 `while (i < j && arr[j] >= pivot) j--`（从右找小于 pivot 者），再执行左侧扫描；若颠倒，最后 `swap(arr[i], arr[low])` 将交换错误的值。A、B、C 关于快排性质的说法均正确。

**第 12 题** D — merge 函数中，处理完左右两半的合并后，需将右侧剩余元素拷贝到 tmp。条件应为 `j <= right`，即右侧指针未越界。A、C 涉及 i，但右侧剩余与 i 无关；B 的 `j < right` 漏掉 j == right 的情况。

**第 13 题** B — 区间调度贪心策略：按结束时间升序排序，每次选最早结束的区间。排序条件 `a[1] < b[1]` 即比较结束时间；选择下一部电影后更新 `lastEnd = movies[i][1]`。A 错：按开始时间排序不能保证最优；C、D 错：更新值应为 lastEnd 而非数组值。

**第 14 题** B — 代码是分治算法（最大子段和的标准分治解法），并非贪心。分治含义：分成左右和跨中间三部分，递归求解后取最大值。A 分治 ✓、C O(n log n) ✓（递推式 T(n) = 2T(n/2) + O(n)）、D 递归 ✓。

**第 15 题** A — 大整数 +1 从末位往前处理：当前位 < 9 则直接 +1 返回；当前位 = 9 则进位，将其置为 0 后继续处理前一位。若所有位均为 9（全进位），循环结束后在开头 insert(1)。B 错：置 9 无法处理进位；C 错：置 1 破坏数值；D 错：数字位不可能为 10。

## 二、判断题

**第 16 题** T — 数学定理：一个数能被 9 整除 ⇔ 其各位数字之和能被 9 整除。代码通过验证 `isDivisibleBy9(n) == isDigitSumDivisibleBy9(n)` 来验算该性质，逻辑正确。

**第 17 题** F — 函数声明返回类型为 `void`，但函数体内执行 `return patternLength`，编译报错。即使修正为返回 int，`lcm(4, 6) = 4×6 ÷ gcd(4,6) = 24 ÷ 2 = 12`，并非 2。

**第 18 题** T — 记忆化递归的斐波那契，每个 n 至多计算一次，后续调用直接返回 memo[n]，时间复杂度 O(n)。无记忆化的朴素递归才是指数级 O(2ⁿ)。

**第 19 题** T — 链表特性：插入删除仅改指针 O(1)（已知位置）；随机访问需遍历 O(n)；每个结点额外存储指针，内存开销大；结点分散在内存各处，对 CPU 缓存不友好。

**第 20 题** F — 二分查找依赖有序性正确，但不仅限于数组。理论上凡支持随机访问的数据结构均可二分，有序链表也可以二分查找（虽需遍历定位，效率 O(n)），并非"仅适用于数组"。

**第 21 题** T — 线性筛通过 `if (i % p == 0) break;` 保证每个合数只被其最小质因子筛到一次，每个合数恰好标记一次，时间复杂度严格 O(n)。

**第 22 题** F — 归并排序稳定（相等元素合并时保持原顺序），但快速排序不稳定（partition 时相等元素相对顺序可能改变）。故"快速排序和归并排序都是稳定的"错误。

**第 23 题** T — 3 柱汉诺塔递归式 T(n) = 2T(n−1) + 1，解得 T(n) = 2ⁿ − 1，时间复杂度 O(2ⁿ)。该代码是分治思想的经典应用。

**第 24 题** T — 所有递归算法理论上均可转化为迭代算法，本质是利用栈（显式或隐式）保存状态。许多递归代码（如汉诺塔）可改为显式栈的迭代写法。

**第 25 题** F — 贪心算法每一步选局部最优，但不保证全局最优。0-1 背包、TSP 等问题中贪心可能得到次优解。贪心需同时满足最优子结构和贪心选择性质才保证全局最优。

## 三、编程题

**第 26 题** 数字选取：问题转化为求 1 加上 [2, n] 中所有质数的个数。理由：1 与任何数互质；大于 1 的数若为合数，必与自身质因子不互质。故最优方案必选取所有质数。使用线性筛（欧拉筛）在 O(n) 内统计质数个数 cnt，输出 1 + cnt 即可。参考程序第 645-651 行正是线性筛标准实现，第 652 行输出 1 + cnt。

**第 27 题** 有趣的数字和：统计 [L, R] 内二进制有奇数个 1 的数之和。用前缀和思想转化为 cal(R) − cal(L−1)。分治函数 `cal(n, p)` 计算 [1, n] 中二进制 1 的奇偶性为 p 的数之和。核心思路：取 n 的最高位 x = 2^k，将区间分为 [1, x−1]（完整 2^k 范围，可用公式直接统计）和 [x, n]（去掉最高位后递归处理）。`cal2` 处理 n = 0 或 1 的边界情况。数据范围达 10⁹，需用 long long（参考程序第 727 行）。
