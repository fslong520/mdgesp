# GESP 四级 2025年06月 真题 解析

---

## 一、选择题

**第 1 题** A — 声明指向整型变量指针的正确语法是 `int* ptr;`。B 用 `*int` 开头、C 将 `*` 放在变量名后、D 语法完全错误，均不合法。

**第 2 题** D — C++ 中二维数组作为函数参数时，可写成 `int arr[3][4]`、`int arr[][4]` 或指针形式 `int (*arr)[4]`，三者等价。`int** arr` 是指向指针的指针，与二维数组的内存布局不兼容，不能接收二维数组。

**第 3 题** D — `int arr[3][4]` 在栈上分配，函数结束自动释放；`new int[12]` 在堆上分配，必须手动 `delete[]` 才能释放，不会自动回收。故 D 说法错误。

**第 4 题** B — `greet()` 的定义在 `main()` 之后，但调用在前且无前置声明。C++ 要求函数在调用前必须有声明或定义，编译器找不到 `greet()` 的声明，报编译错误。

**第 5 题** C — 值传递会复制结构体，函数内修改不影响原变量。引用传递（`struct &s`）和指针传递（`struct *s`）都能直接操作原结构体，达到修改目的。

**第 6 题** B — 形参（parameter）是函数定义中声明的变量名；实参（argument）是调用时传入的具体值。A 说反了，C 忽略了两者本质不同，D 描述不准确。

**第 7 题** C — `print1()` 中局部 `value=50` 遮蔽全局变量，输出 50；`::value` 显式访问全局 `value=100`，输出 100。`print2()` 无局部遮蔽，输出全局 100。结果为 `50 100 100`。

**第 8 题** B — 插入排序的核心思想：维护已排序部分，每次将新元素插入到已排序部分的正确位置。题目描述与此完全吻合。冒泡排序是相邻比较交换，选择排序是每轮选最小值。

**第 9 题** C — 插入排序在数据几乎有序时效率最高，比较和移动次数接近 O(n)。数据量大且乱序严重时应选快排等 O(n log n) 算法；交换次数最少的是选择排序。

**第 10 题** B — 递推算法从已知的基础情况（base case）出发，利用递推关系逐步推导出更大规模问题的解。A 描述的是分治，C 是穷举，D 是贪心策略。

**第 11 题** B — 外层循环遍历 i 从 0 到 n-1（n 次），内层循环遍历 j 从 0 到 n-1（n 次），每次做位判断和累加。两层嵌套均为 n 次，总时间复杂度 O(n²)。

**第 12 题** D — 朴素递归斐波那契中，`fibonacci(n)` 调用 `fibonacci(n-1)` 和 `fibonacci(n-2)`，形成二叉递归树，重复计算大量子问题，时间复杂度为 O(2ⁿ)。

**第 13 题** C — `ifstream in("data.txt")` 以输入模式打开文件，`getline(in, line)` 逐行读取文件内容，`cout << line` 输出到屏幕。程序的功能就是逐行读取并输出文件内容。

**第 14 题** B — try-catch 异常处理机制的核心目的是：在运行时错误发生时，提供结构化的错误处理方式，而非加速程序、编译检查或节省内存。

**第 15 题** B — flag 每轮开始前重置为 false（假设无交换）。内层循环中发生交换则置 flag = true。该轮结束后若 flag 仍为 false，说明已有序，break 退出。故选 B。

## 二、判断题

**第 16 题** T — `int add(int, int);` 是合法的函数声明，指定了返回类型 int 和两个 int 参数。参数名在声明中可以省略。

**第 17 题** F — `foo(int x)` 是值传递，x 是 a 的副本，修改 x 不影响 a。输出应为 10，不是 15。

**第 18 题** F — C++ 允许结构体嵌套定义，即在结构体内部定义新的结构体类型。Library 内嵌套 Book，Book 内嵌套 Author，语法完全正确。

**第 19 题** T — 引用传递直接操作原始变量，无需复制数据。相比值传递（尤其大对象），既避免了拷贝开销，又能修改原变量。

**第 20 题** F — C++ 允许二维数组部分初始化。`{{1,2},{3}}` 中第一行初始化为 {1,2,0}，第二行初始化为 {3,0,0}，未显式赋值的元素自动补 0。代码完全合法。

**第 21 题** F — 循环 `res *= i`，当 i=0 时 `res = 1 * 0 = 0`，此后无论怎么乘结果都是 0。正确写法应为 `res *= (i+1)` 或从 i=1 开始循环。

**第 22 题** T — 选择排序每轮都需遍历未排序区间找最小值，无论数组是否有序，比较次数固定为 n(n-1)/2，即 O(n²)。

**第 23 题** T — 代码是标准选择排序：外层 i 从 0 到 n-2，内层找 [i+1, n) 中最小值下标 minIndex，若与 i 不同则交换。每轮将当前最小值放到位置 i。

**第 24 题** F — 异常未被任何 catch 捕获时，不会在编译时报错，而是在运行时调用 `std::terminate()` 终止程序。编译器不检查运行时异常是否被完全捕获。

**第 25 题** T — `ofstream out("data.txt")` 以输出模式打开文件，`out << "Hello"` 写入字符串，`out.close()` 关闭文件并刷新缓冲区。操作完全正确。

## 三、编程题

**第 26 题 画布裁剪** — 简单的二维数组子矩阵提取。读入 n×m 的字符矩阵和裁剪边界 x1, x2, y1, y2，遍历第 x1 行到第 x2 行、第 y1 列到第 y2 列的子矩阵，逐行输出对应子串。注意题目行列从 1 开始计数，代码中需转换为 0-indexed（即从 `arr[x1-1]` 开始）。时间复杂度 O(n×m)。

**第 27 题 排队** — 本质是求逆序对数量。目标顺序为身高从高到低、体重从重到轻。先按目标顺序排序得到期望序列，再计算原序列到目标序列所需的最少相邻交换次数，即逆序对数。具体做法：定义结构体存储身高体重，按目标规则排序后，对每个原位置找到其在排序后的位置，统计逆序对。也可直接模拟冒泡排序过程计数交换次数。时间复杂度 O(n²)（n≤1000 可接受）。
