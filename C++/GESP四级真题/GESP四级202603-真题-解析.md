## 一、选择题

**第 1 题** B — `f()` 使用默认参数 2，`2 * 3 = 6`；`f(4)` 传入 4，`4 * 3 = 12`。输出 `"6 12"`。

**第 2 题** B — `a = 5`，`p` 指向 `a`，`q` 指向 `p`。`**q += 7` 等价于 `*(*q) += 7` → `*p += 7` → `a += 7`，故 `a = 12`，`*p = 12`。输出 `"12 12"`。

**第 3 题** B — `p` 是行指针，指向每行 4 个 int。`p + 2` 指向第 2 行（0-indexed），`*(p + 2)` 是第 2 行首地址，`*(p + 2) + 1` 是第 2 行第 1 列地址，故 `*(*(p + 2) + 1) = a[2][1] = 10`。

**第 4 题** B — `a` 传值，`x` 不变仍为 1；`b` 传引用，`y += 2` 变为 3；`c` 传指针，`*c += 3` 使 `z` 变为 4。输出 `"1 3 4"`。

**第 5 题** A — `main` 中局部 `x = 10`，`f(x)` 传引用使局部 `x += 2` 变为 12；`::x` 为全局变量 3。输出 `"12 3"`。

**第 6 题** B — C++ 中结构体初始化使用大括号 `{ }` 语法，即 `Point p = {1,2};`。A 是逗号表达式（值为 2）；C 是动态分配语法（返回指针）；D 不是合法语法。

**第 7 题** B — `g(S s)` 传值，`s.a += 10` 不影响原结构体，`s.a` 仍为 1；`h(S& s)` 传引用，`s.b += 10` 使原结构体 `s.b = 12`。输出 `"1 12"`。

**第 8 题** B — 递推算法从已知初值出发，利用递推关系逐步推出后续结果。A 描述的是递归；C 和 D 均错误。

**第 9 题** B — 函数模拟斐波那契数列（初值 `f(1)=1, f(2)=2`），递推计算：
- i=3: c = 1+2 = 3
- i=4: c = 2+3 = 5
- i=5: c = 3+5 = 8
- i=6: c = 5+8 = 13
返回 13。

**第 10 题** B — 排序结果中相同 score（90）的元素保持原顺序 A→B→D，说明算法对相等关键字是稳定的。C 的 score 为 80（最小），自然应排在最前，与稳定性无关。

**第 11 题** B — 插入排序升序时，内层循环将比 `key` 大的元素依次后移，故条件为 `a[j] > key`。

**第 12 题** C — 外层循环 n 次，内层循环 n 次，双层循环总执行次数为 n² 数量级，时间复杂度 O(n²)。

**第 13 题** B — `cout` 是标准输出流对象，不能通过 `ofstream` 直接重定向。选项 B 创建了文件输出流但未改变 `cout` 的流对象，故无法将 `cout` 的输出写入文件。A 使用 `freopen` 重定向标准输出；C 使用 `rdbuf` 替换 cout 的缓冲区；D 直接使用 `outFile` 输出到文件。

**第 14 题** B — `divi(10, 0)` 抛出 `int` 类型的值 0，被 `catch(int)` 捕获，输出 `"B"`。`catch(const char*)` 不匹配 int 类型。

**第 15 题** A — 玩家分数增加后需向前移动，排行榜通常是降序排列（高分在前）。`while` 条件应为：仍在有效范围内（`i > 0`）且当前分数大于前一个玩家分数（`cur.score > players[i-1].score`），则继续前移。

## 二、判断题

**第 16 题** T — `add10(int &x)` 使用引用传参，`x` 是 `a` 的别名，`x += 10` 直接修改 `a`，故 `a = 15`。

**第 17 题** T — 引用在初始化时绑定一个变量后，不能更改绑定到其他变量，这是引用的核心特性之一。

**第 18 题** F — `a[2][3]` 是 2 行 3 列的二维数组。`&a[1][2]` 和 `&a[0][1]` 的指针差值计算：`(1 × 3 + 2) - (0 × 3 + 1) = 5 - 1 = 4`，输出为 4，不是 5。

**第 19 题** F — `calc(int x, int y = 10)` 是声明（带默认参数），但后续重载了 `calc(int x)` 和 `calc(int x, int y)`。调用 `calc(5)` 时，编译器无法区分是匹配 `calc(int x)` 还是 `calc(int x, int y = 10)`，造成二义性，编译错误。

**第 20 题** T — `f()` 中局部变量 `x = 20` 覆盖全局变量，输出 `"20"`；回到 `main` 后输出全局 `x = 10`。故输出 `"2010"`。

**第 21 题** F — C++ 中局部作用域的指针变量不会自动初始化，其值为未定义的随机地址（野指针）。只有全局或静态指针会被自动初始化为 `nullptr`。

**第 22 题** T — 结构体内可以嵌套定义其他结构体，语法正确。`GameCharacter` 内部定义了 `Equipment` 结构体和 `Skill` 结构体，并声明了相应成员变量，符合 C++ 语法规范。

**第 23 题** F — `cout << "Hello"` 将字符串输出到标准输出（屏幕），而非 `fout` 关联的文件。应将 `cout` 改为 `fout` 才能写入文件。

**第 24 题** F — 虽然两者最坏时间复杂度均为 O(n²)，但插入排序在数据近乎有序时最佳时间复杂度为 O(n)，而选择排序始终为 O(n²)，实际性能差异显著，不能简单互相替代。

**第 25 题** F — 该程序使用递推（迭代）方式计算斐波那契数，仅用一层循环，时间复杂度为 O(n)。O(2ⁿ) 是递归方式（不记忆化）的时间复杂度。

## 三、编程题

**第 26 题 山谷** — 遍历网格中每个单元格，检查其 8 个方向（上、下、左、右、左上、右上、左下、右下）的相邻单元格。若该单元格海拔不高于所有相邻单元格，则计数加一。注意边界处理，行首、行尾、列首、列尾的单元格相邻方向较少。时间复杂度 O(n × m)。

**第 27 题 礼盒排序** — 对每个礼盒计算总价、最大值、最小值，存入结构体数组。按照题目给定的四级排序规则（总价 → 最大值 → 最小值 → 编号）使用 `sort` 自定义 `cmp` 函数进行排序。输出排序后的编号。时间复杂度 O(n × k + n log n)。
