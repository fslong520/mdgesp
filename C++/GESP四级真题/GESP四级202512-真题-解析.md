## 一、选择题

**第 1 题** C — 声明 int 指针应写作 `int *p = &x;`。A 将 int* 赋给 int，类型不匹配；B 将 int 值赋给指针，错误；D 语法错误。

**第 2 题** C — `p2 = p1` 使 p2 指向与 p1 相同的地址（即 a）。`*p2 = 10` 通过 p2 修改 a 为 10，故 a=10, *p1=10, *p2=10。

**第 3 题** B — 二维数组访问使用 `a[行][列]` 语法。第 2 行第 3 列即 `a[1][2]`（0-indexed）。A 是逗号表达式（结果为 a[2]）；C、D 语法错误。

**第 4 题** D — `p` 指向 `arr[0][0]`，`*(p + 5)` 即往后偏移 5 个 int，即 `arr[0][5]` → 一行 4 个元素，所以是 `arr[1][1] = 6`。`arr[1][1]` 也是 6，故两者均为 6。

**第 5 题** B — 三重循环累加 `(i+j+k) % 2 == 0` 的元素：
- i=0,j=0,k=0(1) + i=0,j=1,k=1(4) + i=0,j=2,k=0(5) + i=1,j=0,k=1(8) + i=1,j=1,k=0(9) + i=1,j=2,k=1(12) = 1+4+5+8+9+12 = 39。

**第 6 题** A — test() 内部：块内 a=3→4（局部），块外 a=2→3（test 的局部）。输出 `3 `。主函数输出全局 a=1。故 `3 1`。

**第 7 题** C — `byValue(a)` 传值不改变原变量（a=1）；`byRef(b)` 传引用使 b=200；`byPointer(&c)` 传指针使 c=300。

**第 8 题** A — 初始化后 topLeft=(10,10), bottomRight=(20,20)。`rect.topLeft.x = 5` 使 topLeft.x=5；`p->y = 5` 使 bottomRight.y=5。输出 5+5=10。

**第 9 题** B — 类似斐波那契数列，初值 f(1)=1, f(2)=2。递推：i=3→3, i=4→5, i=5→8。故返回 8。

**第 10 题** B — 原顺序中 value=3 的元素为 {3,'B'} 在前、{3,'D'} 在后。排序后 {3,'D'} 出现在 {3,'B'} 之前，相对顺序改变，故不稳定。

**第 11 题** B — 选择排序找前 k 大，每次从未排序区间 `[i+1, n)` 中找到最大值的下标，故内层循环为 `int j = i + 1; j < n; j++`。

**第 12 题** A — 分数增加需向前移动（降序排列），条件为 `i > 0 && updatedPlayer.score > players[i-1].score`。分数减少需向后移动，条件为 `i < size-1 && updatedPlayer.score < players[i+1].score`。

**第 13 题** D — 外层循环 n 次，内层循环 n 次，每次检查 i 的第 j 位做条件累加，总时间复杂度为 O(n²)。

**第 14 题** D — `fin >> s1` 读取 "Happy"（遇到空格/换行停止），光标停在换行符后。`getline(fin, s2)` 读取剩余内容，但此时光标位置紧接着换行符，`getline` 读取到换行即结束，故 s2 为空字符串。输出 `"Happy|"`。

**第 15 题** B — `divide(10, 0)` 抛出 `const char*` 类型的字符串，被 `catch(const char* msg)` 捕获，输出 "B"，result 赋值为 -1，然后输出 -1，故输出 `"B-1"`。

## 二、判断题

**第 16 题** T — `*p = *p + 3` 等价于 `x = x + 3 = 5 + 3 = 8`，故 x 变为 8。

**第 17 题** F — C++ 中结构体可以包含另一个结构体作为成员变量（嵌套结构体），语法允许。

**第 18 题** T — C++ 二维数组按行优先（row-major）连续存储，同一行的元素在内存中地址连续。

**第 19 题** T — `add(int &x)` 使用引用传参，x 是 a 的别名，`x += 10` 直接修改 a，故 a = 15。

**第 20 题** F — 指针相减得到的是两个指针之间元素的个数差（即 `(arr+2) - arr = 2`），而非字节数差。输出为 2，不是 8。

**第 21 题** F — 使用递推（迭代）方式计算，仅一层循环，时间复杂度为 O(n)。O(2ⁿ) 是递归不记忆化的时间复杂度。

**第 22 题** T — 冒泡排序和插入排序在比较相等元素时不会交换位置，因此都是稳定的排序算法。

**第 23 题** F — 该代码实现的是**插入排序**（从 i=1 开始，将 a[i] 插入到已排序部分），而非选择排序。选择排序是每次找最小值放到前面。

**第 24 题** F — `calculate(int x, int y = 10)` 是带默认参数的声明，后面重载了 `calculate(int x)` 和 `calculate(int x, int y)`。调用 `calculate(5)` 时，两个函数均可匹配（一个直接匹配，一个使用默认参数），造成二义性，编译错误。

**第 25 题** T — 文件写入 10, 20, 30, 40 四个数。`fin >> a >> b >> c >> d` 依次读取，得 a=10, b=20, c=30, d=40，求和为 100。

## 三、编程题

**第 26 题 建造** — 枚举所有 3×3 子区域（左上角遍历范围 i: 1→M-2, j: 1→N-2），对每个区域求最大值、最小值、总和。若 max-min ≤ H，则更新答案。时间复杂度 O(M×N)。

**第 27 题 优先购买** — 定义结构体存储商品信息（name, price, priority）。按优先级（小→大）、价格（小→大）、名字（字典序）排序。遍历排序后的商品，若价格 ≤ 剩余预算则购买，预算减去价格。最后将购买的商品名按字典序排序输出。
